# Task T-043@20260515T214233Z: PAL Deploy Rollback CLI Approval Gate

## Problem Statement

PAL-033 requires an operator-facing rollback command:

```bash
promptclaw pal deploy rollback PROJECT_ROOT --remote-inventory PATH --backup-id ID --approve-rollback
```

The repo already has a local fake-remote rollback primitive,
`promptclaw.pal_deploy.rollback_pal_deployment_backup(...)`, but there is no
CLI parser or dispatch path for it. Operators therefore cannot exercise the
rollback path through the same approval-gated `promptclaw pal deploy` surface
used by `plan` and `apply`.

The command must refuse rollback without `--approve-rollback` and must avoid
live SSH deployment, service restarts, provider secrets, dependencies,
migrations, and database changes.

## Technical Approach

- Extend `promptclaw.cli.build_parser()` under `pal deploy` with a `rollback`
  subcommand.
- Match existing deploy CLI style:
  - required `PROJECT_ROOT`
  - optional `--manifest`
  - required `--remote-inventory`
  - required `--backup-id`
  - boolean `--approve-rollback`
  - optional `--json`
- Add `cmd_pal_deploy_rollback(args)` that:
  - resolves project-relative paths consistently with `plan` and `apply`
  - refuses immediately when `args.approve_rollback` is false
  - emits a JSON/human rejected payload with `remote_writes=false`,
    `live_ssh=false`, and `service_restarts=false`
  - loads the backup artifact from
    `PROJECT_ROOT/.promptclaw/pal-deploy/backups/<backup-id>/backup-manifest.json`
  - loads the supplied fake remote inventory snapshot
  - calls `rollback_pal_deployment_backup(..., approved=True)`
  - prints JSON or concise human output
- Add a small typed backup-manifest loader/formatter in `promptclaw.pal_deploy`
  if needed, rather than duplicating ad hoc JSON parsing in the CLI.
- Dispatch `pal deploy rollback` through `_dispatch_pal`.
- Update PAL deployment docs and changelog to replace "future rollback CLI"
  language with the approved fake-remote rollback command.

## Edge Cases

- Missing `--approve-rollback`: returns nonzero, writes no backup files, and
  does not mutate the fake remote inventory.
- Missing backup manifest: command returns nonzero through existing diagnosis
  error handling.
- Missing backup file referenced by the backup manifest: primitive raises
  `PALDeploymentManifestError`, preserving the inventory.
- Backup sha256 mismatch: primitive raises before writing the inventory.
- Project-relative `--manifest` and `--remote-inventory` resolve under
  `PROJECT_ROOT`; absolute paths are preserved.
- Unmanaged fake-remote inventory entries remain untouched by rollback.
- Startup identity hardening is not changed by this task; existing startup
  identity regression tests remain mandatory verification anchors.
