# Specification: T-022@20260416T043834Z

## Problem Statement
CypherClaw's SenseWeave requires abstractions to manage real-time environmental input (live buffers) and long-term retention of notable audio events (sample banks). We need a stable representation of the current audio buffer and a mechanism to persistently archive valuable clips with associated metadata.

## Technical Approach
1. Create a new module `my-claw/tools/senseweave/live_buffer.py`.
2. Define a `LiveBuffer` class that exposes `source_name`, `capture_path`, `freshness`, `rms`, `spectral_profile`, and `transform_history`. Provide a method to refresh this state from new sensory data.
3. Define a `SampleBankStorage` (or `SampleArchive`) class representing the "sample bank" abstraction. It will handle copying/moving selected clips to an archive directory and storing associated metadata (e.g., as JSON files side-by-side with the audio).
4. Create comprehensive tests in `tests/test_live_buffer.py` to ensure `LiveBuffer` state updates correctly upon refresh, and that `SampleBankStorage` accurately selects archive paths and persists metadata.

## Edge Cases
- Missing source files during retention (should gracefully fail or return `None`).
- Empty spectral profiles or missing RMS values on refresh (should default to zeros or retain previous valid values based on the semantics).
- Metadata persistence formatting and safety.

## Acceptance Criteria
- stable buffers expose source name, capture path, freshness, RMS, spectral profile, and transform history
  - VERIFY: `grep -q "class LiveBuffer" my-claw/tools/senseweave/live_buffer.py`
- sample banks can retain selected clips on archive storage with metadata
  - VERIFY: `grep -q "def retain_clip" my-claw/tools/senseweave/live_buffer.py`
- tests verify buffer refresh and archive path selection
  - VERIFY: `pytest tests/test_live_buffer.py -x`
