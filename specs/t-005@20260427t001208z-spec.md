# Task T-005: Live smoke test against real Replicate

## Problem Statement
The Replicate generation backend currently has unit-level retry coverage
(`tests/test_generation_replicate_retry.py`) and a parallel live smoke
suite for Modal (`tests/test_generation_live_modal.py`), but no
end-to-end smoke test exists that exercises `ReplicateClient.generate(req)`
against the real Replicate API. We need a gated live test that, when
opted into, proves the production submission/polling/download path
returns a real WAV, records non-zero cost, and lands inside a sane
latency budget — so we can catch regressions in the live integration
before they reach the composer.

## Technical Approach
- Add `tests/test_generation_live_replicate.py` paralleling the Modal
  live test in shape and import strategy (insert `my-claw/tools` onto
  `sys.path`, then import `ReplicateClient`).
- Mark the single test `@pytest.mark.live_replicate` so the default
  `pytest tests/` run skips it.
- Extend `tests/conftest.py` with a sibling `--run-live-replicate`
  CLI flag, a registered `live_replicate` marker, and the matching
  collection-time skip when the flag is absent.
- Inside the test, additionally skip when the `REPLICATE_API_TOKEN`
  environment variable is unset so the test cannot accidentally fail
  in a CI environment that opts into the flag without a token.
- Use a fixed deterministic prompt and seed so the run is reproducible
  across operators: `prompt="ambient pad, slow, sparse, 30 seconds"`
  and `seed=42` (mirrors the Modal live test seed).
- After calling `client.generate(request)`, assert the full acceptance
  contract: WAV file (or bytes) larger than 100 KB, `cost_usd` between
  `$0.01` and `$0.50`, and `latency_ms < 60000`.
- Document in the test docstring that each run costs roughly $0.10
  against the real Replicate API.
- The implementation of `ReplicateClient._submit_prediction` itself is
  out of scope for T-005 and lands under T-001 (currently
  `needs_split`). The live test is therefore expected to be skipped by
  default and only runnable end-to-end once T-001 ships the real
  submission/polling/download body. This is consistent with how the
  Modal live test landed under T-003 alongside the still-stub-tier
  protocol parity tests.

## Edge Cases
- `REPLICATE_API_TOKEN` set but `--run-live-replicate` not passed: test
  is skipped at collection time via the conftest hook.
- `--run-live-replicate` passed but `REPLICATE_API_TOKEN` unset: the
  test body skips with `pytest.skip(...)` so no spurious failures.
- WAV is delivered either as bytes or a downloaded file path; the
  assertion logic accepts either shape and measures total payload size.
- The result mapping must expose `cost_usd` and `latency_ms` keys to
  parallel the Modal client's contract; if the eventual T-001 result
  shape is a dataclass instead of a dict, the test reads attributes via
  `getattr`-style access in the same way (we use a small helper to
  normalize the lookup).
- The recurring hardening anchors about `bootstrap_identity()`,
  resource-governor budget enforcement, and `revise_score()` wiring are
  outside this generation-client smoke test; the existing tests in
  `tests/test_first_boot.py`, `tests/test_governor_integration.py`,
  `tests/test_resource_governor.py`, and `tests/test_self_critique.py`
  remain the verification anchors and will be re-run during validation.

## Acceptance Criteria
- `tests/test_generation_live_replicate.py` exists and is decorated
  `@pytest.mark.live_replicate`.
  - **VERIFY:** `grep -q "@pytest.mark.live_replicate" tests/test_generation_live_replicate.py`
- The test runs only when both `REPLICATE_API_TOKEN` is set and
  `--run-live-replicate` is passed; otherwise it is skipped.
  - **VERIFY:** `pytest tests/test_generation_live_replicate.py -v -m "not live_replicate"` (passes by collecting zero or skipping the test)
  - **VERIFY:** `pytest tests/test_generation_live_replicate.py -v` (skips because the flag is absent)
- The test calls `ReplicateClient.generate(req)` with a fixed prompt
  and seed.
  - **VERIFY:** `grep -E "ReplicateClient\\(\\)|client\\.generate\\(" tests/test_generation_live_replicate.py`
- The test asserts that the result represents a real WAV file with
  payload size greater than 100 KB.
  - **VERIFY:** `grep -E "100[_ ]?\\* ?1024|> 100" tests/test_generation_live_replicate.py`
- The test asserts `cost_usd > 0.01 and cost_usd < 0.50`.
  - **VERIFY:** `grep -q "0.01" tests/test_generation_live_replicate.py && grep -q "0.50" tests/test_generation_live_replicate.py`
- The test asserts `latency_ms < 60000`.
  - **VERIFY:** `grep -q "60000" tests/test_generation_live_replicate.py`
- The test docstring records the approximate per-run cost (~$0.10).
  - **VERIFY:** `grep -E "\\$0\\.10|~\\$0.10" tests/test_generation_live_replicate.py`
- `conftest.py` registers the `live_replicate` marker and
  `--run-live-replicate` flag and skips marked tests when the flag is
  absent.
  - **VERIFY:** `grep -q "run-live-replicate" tests/conftest.py && grep -q "live_replicate" tests/conftest.py`
- Default `pytest tests/` run is unaffected (the live test is skipped).
  - **VERIFY:** `pytest tests/ -x`
- Lint and types remain clean.
  - **VERIFY:** `ruff check src/ tests/ && mypy src/`
