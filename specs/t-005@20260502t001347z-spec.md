# Task T-005@20260502T001347Z: Pytest test suite for the narrative API

## Problem Statement

The CypherClaw narrative HTTP service now exposes `POST /memory/search`,
`GET /events`, and `POST /events` (T-001/T-002/T-003 of this sprint). T-005
locks in the regression coverage we want before later sprints (T-007
domain-column migration, T-010 shared-secret auth, T-011+ world/beats
endpoints) keep widening the surface. Every existing endpoint needs at least
the four scenarios named in the task graph:

1. success path,
2. auth failure (when auth is enabled),
3. bad input validation,
4. downstream-engine error handling.

Coverage on the `cypherclaw.narrative_api` module must be ≥80%, and the wire
acceptance criterion explicitly counts ≥1 test per endpoint.

## Technical Approach

- Keep tests in the existing `tests/test_narrative_api_*.py` files (this
  repo's `pyproject.toml` pins `testpaths = ["tests"]`; the task graph's
  `cypherclaw/src/cypherclaw/narrative_api/tests/` path describes the
  deploy-side mirror, not this PromptClaw queue checkout).
- Add a tiny `auth_token: str | None` keyword to `create_app(...)`. When set,
  every endpoint requires the `X-Narrative-Auth` request header to match;
  missing/bad headers return `401`. When unset, behavior is unchanged so the
  existing tests stay green and so this slice does not pre-empt T-010's
  env-var integration / startup warning log.
- For each endpoint, add an "auth failure when enabled" test that builds an
  app with `auth_token="secret"`, sends a request without (or with the wrong)
  `X-Narrative-Auth` header, and asserts `401`.
- For each endpoint, add an "auth success when enabled" test that sends the
  correct header and asserts the request still routes to the underlying
  store/searcher exactly once — proving the gate did not swallow downstream
  calls.
- Add the missing scenarios on `GET /events` to bring it up to the required
  4-scenario floor:
  - bad input validation: out-of-range `limit` returns 422 before the store
    is invoked,
  - the existing downstream-failure test already covers the 502 path.
- Cover the lazy-import / `503` fallback paths in `events.py` and `memory.py`
  so the coverage floor is met without inflating the implementation surface:
  - `build_default_event_store()` and `build_default_memory()` raise
    `EventStoreUnavailableError` / `MemoryUnavailableError` when the
    deploy-side packages are absent (the default in this checkout).
  - The app wraps those errors as HTTP `503` for each endpoint when no
    injection is provided — exercise that path through `TestClient`.
- Cover `_normalize_event` / `_normalize_result` shape branches that lift
  coverage cheaply: dataclass / object-shaped events from append, tuple-shaped
  memory hits with metadata domains, distance-only similarity scoring.
- Run `pytest --cov=cypherclaw.narrative_api` once at the end to verify the
  ≥80% floor; `pytest-cov` is already installed in this environment.
- Do not modify the request/response Pydantic schemas, the deploy-time lazy
  import, or the existing T-010 design. Auth wiring stays intentionally
  minimal: a single dependency, no env-var coupling, no warning log.

## Edge Cases

- `auth_token=""` is treated as auth-disabled (matches the spirit of T-010's
  "unset → disabled" rule and avoids accidentally locking out tests that
  pass an empty string by mistake).
- `X-Narrative-Auth` header comparison is exact-match `str == str`; we do not
  lowercase, strip, or constant-time-compare in this slice (T-010 owns the
  hardened comparison and the env-var pull).
- A correct `X-Narrative-Auth` header must continue to allow Pydantic
  validation errors (422) and downstream `502`s through; auth must be
  evaluated before the request body but must not swallow later errors.
- `GET /events` with `limit=0` or `limit=2000` returns 422 from Pydantic
  before the event store is touched (existing FastAPI `Query(ge=1, le=1000)`
  bound).
- `build_default_*` raises in this checkout because no deploy-side
  `cypherclaw.narrative.{memory,events}` package is installed; tests assert
  the resulting `503` so the lazy-import branch stays covered when later
  tasks land migrations that change import behavior.
- The recurring hardening anchors (`bootstrap_identity()` startup, resource
  governor, `revise_score()` self-critique) are unrelated to the narrative
  HTTP surface; the existing `tests/test_first_boot.py`,
  `tests/test_governor_integration.py`, and `tests/test_self_critique.py`
  remain the regression anchors and are re-run during validation.

## Acceptance Criteria

- Each narrative endpoint has ≥1 success-path test, ≥1 auth-failure test
  with auth enabled, ≥1 bad-input-validation test, and ≥1 downstream-engine
  error test.
  - **VERIFY:** `pytest tests/test_narrative_api_memory.py tests/test_narrative_api_events.py -q`
- `POST /memory/search` rejects requests with a missing/wrong
  `X-Narrative-Auth` header when `auth_token` is configured.
  - **VERIFY:** `pytest tests/test_narrative_api_memory.py::TestMemorySearchEndpoint::test_auth_enabled_missing_header_returns_401 tests/test_narrative_api_memory.py::TestMemorySearchEndpoint::test_auth_enabled_wrong_token_returns_401 -q`
- `POST /memory/search` succeeds with a matching `X-Narrative-Auth` header
  when `auth_token` is configured.
  - **VERIFY:** `pytest tests/test_narrative_api_memory.py::TestMemorySearchEndpoint::test_auth_enabled_correct_header_succeeds -q`
- `GET /events` and `POST /events` reject missing/wrong auth headers when
  `auth_token` is configured and accept matching headers.
  - **VERIFY:** `pytest tests/test_narrative_api_events.py::TestEventsEndpoint::test_auth_enabled_missing_header_returns_401 tests/test_narrative_api_events.py::TestEventsEndpoint::test_auth_enabled_correct_header_succeeds tests/test_narrative_api_events.py::TestEventsAppendAuth::test_auth_enabled_missing_header_returns_401 tests/test_narrative_api_events.py::TestEventsAppendAuth::test_auth_enabled_correct_header_succeeds -q`
- `GET /events` rejects out-of-range `limit` (validation) before the store
  is touched.
  - **VERIFY:** `pytest tests/test_narrative_api_events.py::TestEventsEndpoint::test_invalid_limit_rejected_before_store -q`
- When no event store / memory searcher is injected and the deploy-side
  package is absent, every endpoint returns HTTP `503` with the unavailable
  detail.
  - **VERIFY:** `pytest tests/test_narrative_api_memory.py::TestMemorySearchEndpoint::test_default_searcher_missing_returns_503 tests/test_narrative_api_events.py::TestEventsEndpoint::test_default_store_missing_returns_503 tests/test_narrative_api_events.py::TestEventsEndpoint::test_post_default_store_missing_returns_503 -q`
- Module coverage on `cypherclaw.narrative_api` is ≥80%.
  - **VERIFY:** `pytest --cov=cypherclaw.narrative_api --cov-report=term-missing tests/test_narrative_api_memory.py tests/test_narrative_api_events.py`
- Existing startup identity / governor regression anchors remain green.
  - **VERIFY:** `pytest tests/test_first_boot.py tests/test_governor_integration.py -q`
- Required validation remains clean.
  - **VERIFY:** `pip install -e '.[dev]' && pytest tests/ -x && ruff check src/ tests/ && mypy src/`
