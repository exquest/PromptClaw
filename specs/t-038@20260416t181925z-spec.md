# Specification: T-038@20260416T181925Z

## Problem Statement

`compose_score_tree()` creates `SectionNode` instances with default-constructed
`SectionEnvelope()` — all parameters at flat defaults. The procedural arc system
already computes rich per-section metadata (density, dynamic, harmonic target,
spatial intent, synthesis type) via `ArcDirective`, and the piece brief carries
narrative state (arc_phase, mood, creative_energy, curiosity). None of this
narrative context currently flows into the section envelope, so tension
trajectories, brightness curves, and density targets are disconnected from the
dramatic arc.

This task couples the narrative engine (procedural arc + piece brief narrative
state) to SectionEnvelope selection so dramatic arcs drive section choice.

## Technical Approach

### New module: `src/cypherclaw/render/narrative_envelope.py`

1. Define `ToneVector` (frozen dataclass):
   - `darkness`: float [0.0, 1.0] — maps from mood (negative mood → higher darkness)
   - `energy`: float [0.0, 1.0] — maps from creative_energy
   - `warmth`: float [0.0, 1.0] — derived from timbre target
   - `density`: float [0.0, 1.0] — from arc directive density_target

2. Define `NarrativeBeat` (frozen dataclass):
   - `arc_phase`: str — phase name (Divination/Emergence/Conversation/Convergence/Crystallization)
   - `section_function`: str — from form grammar (invocation/statement/development/turn/recap/coda/etc.)
   - `tension_target`: float [0.0, 1.0] — overall tension level for this beat
   - `tone`: ToneVector — tone dimensions for this beat

3. Implement `envelope_for_beat(beat: NarrativeBeat) -> SectionEnvelope`:
   - **tension_trajectory**: breakpoint function shaped by arc_phase (each phase
     has a characteristic shape: Divination rises slowly, Conversation sustains
     high, Crystallization settles). Section function modifies the shape
     (development/turn push the peak, recap/coda pull toward resolution).
   - **density_target**: scalar from `tone.density`, clamped to [0.05, 0.95].
   - **dynamic_plane**: derived from `tone.energy` with `tone.darkness` influence.
     Higher energy → higher dynamic plane; higher darkness → slight reduction.
   - **brightness**: derived from `1.0 - tone.darkness` with energy boost.
   - **tempo_base**: scaled from a base of 1.0, modified by arc phase energy
     (Conversation faster, Crystallization slower).

4. Implement `tone_from_narrative(*, arc_phase, mood, creative_energy, curiosity,
   density_target, dynamic_target, timbre_target) -> ToneVector`:
   - Converts raw narrative state into a normalized ToneVector.

5. Implement `beat_from_directive(directive, section_function) -> NarrativeBeat`:
   - Convenience factory that builds a NarrativeBeat from an ArcDirective +
     section function string.

### Integration in `recursive_composer.py`

6. Replace `SectionEnvelope()` on line 235 with:
   ```python
   section_envelope=envelope_for_beat(
       beat_from_directive(directive, section.function)
   )
   ```
   where `directive` is the per-section ArcDirective already computed in
   `_production_arc_for_sections()`. Refactor to return directives alongside
   metadata so they're available during section construction.

## Edge Cases

- Missing arc directive (commission.arc_directive is None): fall back to
  Emergence-phase defaults for the tone vector.
- Unknown arc phase name: treat as Emergence (the middle-ground phase).
- Unknown section function: use neutral tension modifiers (no boost or reduction).
- All parameters must remain within their validated ranges after scaling.

## Assumptions

- The "existing narrative engine interface" is the procedural arc system
  (ArcDirective, ArcPhase, directive_for_elapsed) since the full narrative
  engine from the PRD is not yet built.
- The ToneVector here is a lightweight musical-facing tone representation,
  not the full 8-dimension NE-009 ToneController — that will be built when
  the narrative engine PRD is implemented. This module is designed to be
  replaced/extended by the full tone controller later.
- No new external dependencies are required.
- The candidate hardening items referencing bootstrap_identity,
  FirstBootAnnouncer, budget enforcement, and revise_score integration are
  out of scope for this task (narrative envelope coupling has no interaction
  with startup sequencing, resource budgets, or the self-critique pipeline).
