# Task T-032b: Extract steady-state cost data for the last 7+ days

## Problem Statement

The new `sdp/telemetry/cost-model.md` reference (T-032@a) documents how SDP
prices token usage, but operators have no rolled-up cost view. The
`telemetry` table in `.sdp/state.db` already records per-task `input_tokens`,
`output_tokens`, `cache_creation_tokens`, `cache_read_tokens`, and the
`lead_profile_id` of the agent that produced each row, so the data needed
for a steady-state cost picture exists. We need a small extractor and a
checked-in CSV that summarizes the last 7+ active days of SDP runs by
applying the cost-model.md pricing to those telemetry rows.

## Technical Approach

- Add `promptclaw/sdp_cost.py` exposing:
  - `TOKEN_PRICING` — ordered tuple of `(pattern, input_per_m, output_per_m)`
    rows mirroring the cost-model.md table, including the `fallback`
    sentinel.
  - `match_pricing(profile_id: str) -> tuple[str, float, float]` —
    case-insensitive substring match against `TOKEN_PRICING`; falls back
    to the `fallback` row when nothing matches.
  - `call_cost_usd(input_tokens: int, output_tokens: int, profile_id: str) -> float`
    applying `(input/1e6 * input_per_m) + (output/1e6 * output_per_m)`.
  - `DailyCostRow` dataclass with `date`, `runs`, `input_tokens`,
    `output_tokens`, `cache_creation_tokens`, `cache_read_tokens`,
    `total_cost_usd`, `models` (sorted unique pricing tiers seen).
  - `extract_steady_state_cost(db_path: Path, days: int = 7) -> list[DailyCostRow]`
    selecting the `days` most-recent distinct UTC `date(created_at)` values
    from `telemetry`, aggregating one row per day, ordered ascending.
  - `write_csv(rows: list[DailyCostRow], out_path: Path) -> None` writing
    a stable header row plus one CSV row per day.
- Add `python -m promptclaw.sdp_cost` CLI thin wrapper supporting
  `--db`, `--days`, and `--out` (defaults: `.sdp/state.db`, `7`,
  `sdp/telemetry/steady-state-cost.csv`).
- Run the CLI once and commit the resulting CSV at
  `sdp/telemetry/steady-state-cost.csv`. The CSV is the human-readable
  artifact required by the task title; the module is its source of truth.

## Edge Cases

- The current `telemetry` table contains 7 distinct active days
  (2026-04-08, 04-09, 04-16, 04-17, 04-25, 04-26, 04-27) with calendar
  gaps. "Last 7+ consecutive days" is therefore interpreted as the 7
  most recent **active** days; the CSV groups by `date(created_at)` and
  the calendar gaps are visible from the date column.
- `lead_profile_id` patterns in current data are `claude-opus-4-7-*`,
  `claude-opus-4-6-*`, `claude-sonnet-4-6-*`, `gpt-5.4-*`, `gpt-5.5-*`,
  `gemini-3.1-*`. Per cost-model.md substring rules,
  `claude-opus-4-*` matches `claude-opus-4`, `claude-sonnet-4-*` matches
  `claude-sonnet-4`, and the rest fall through to `fallback` ($3/$15).
- Empty `lead_profile_id` falls through to `fallback`.
- Zero or missing token counts yield `0.0` cost without raising.
- A telemetry row's `input_tokens` / `output_tokens` are billed against
  the row's `lead_profile_id` only; verify-agent token splits are not
  separately recorded in the table and are therefore folded into the
  same row, matching the existing cost-summary contract.
- `cache_creation_tokens`, `cache_read_tokens`, and `thinking_tokens`
  are reported as columns but, per cost-model.md, do not contribute to
  `total_cost_usd`.

## Acceptance Criteria

1. `promptclaw.sdp_cost` exposes `TOKEN_PRICING`, `match_pricing`,
   `call_cost_usd`, `DailyCostRow`, `extract_steady_state_cost`, and
   `write_csv` with the documented signatures and applies the
   cost-model.md pricing table verbatim.
   - **VERIFY:** `pytest tests/test_sdp_cost_extractor.py -q`

2. `match_pricing` is a case-insensitive substring match that returns
   the cost-model.md tier names (`claude-opus-4`, `claude-sonnet-4`,
   `claude-haiku`, `gpt-4o`, `o3`, `codex-mini`, `fallback`) and
   `call_cost_usd` matches the cost-model.md formula
   (`(in/1e6*in_p) + (out/1e6*out_p)`).
   - **VERIFY:** `pytest tests/test_sdp_cost_extractor.py::test_match_pricing_substring_and_fallback tests/test_sdp_cost_extractor.py::test_call_cost_usd_matches_cost_model_formula -q`

3. `extract_steady_state_cost(db_path, days=7)` returns one
   `DailyCostRow` per UTC date for the most recent `days` distinct
   active days in the `telemetry` table, ordered ascending by date,
   with per-day totals for runs, input/output/cache tokens, USD cost,
   and the sorted unique pricing tiers seen.
   - **VERIFY:** `pytest tests/test_sdp_cost_extractor.py::test_extract_steady_state_cost_groups_and_prices_by_day -q`

4. `write_csv` writes a stable header
   (`date,runs,input_tokens,output_tokens,cache_creation_tokens,cache_read_tokens,total_cost_usd,models`)
   followed by one row per day, with `total_cost_usd` formatted to four
   decimal places and `models` joined by `|`.
   - **VERIFY:** `pytest tests/test_sdp_cost_extractor.py::test_write_csv_emits_stable_schema -q`

5. The CLI `python -m promptclaw.sdp_cost --db <db> --days 7 --out <csv>`
   writes the CSV to the requested path and prints the path on stdout.
   - **VERIFY:** `pytest tests/test_sdp_cost_extractor.py::test_cli_writes_csv_and_prints_path -q`

6. `sdp/telemetry/steady-state-cost.csv` is committed and contains 7
   data rows derived from the live `.sdp/state.db` snapshot.
   - **VERIFY:** `python -c "import csv,pathlib; rows=list(csv.reader(pathlib.Path('sdp/telemetry/steady-state-cost.csv').open())); assert rows[0][0]=='date' and len(rows)==8"`

7. Product-facing change notes mention the new steady-state cost CSV
   and extractor.
   - **VERIFY:** `grep -n "steady-state-cost.csv" CHANGELOG.md`
