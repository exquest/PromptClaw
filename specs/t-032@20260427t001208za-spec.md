# Task T-032a: Document SDP telemetry cost model

## Problem Statement

PromptClaw's SDP cost assumptions are currently distributed across telemetry summaries, model pricing helpers, token prediction, capacity/admission logic, and generation budget code. Operators need one Markdown reference under `sdp/telemetry/` that enumerates token/per-call costs, fixed overhead assumptions, and cap formulas without requiring source-code spelunking.

## Technical Approach

- Add `sdp/telemetry/cost-model.md` as the single reference for current cost accounting.
- Document SDP token billing from telemetry fields and the static pricing table used by `sdp.pipeline.cost_summary`.
- Document capacity/admission cap formulas from token prediction and capacity code, including reserve behavior and safe concurrency.
- Document CypherClaw generation per-call cost estimates because generation budget code supplies the current fixed-overhead/per-request model.
- Add static regression tests that lock the expected headings, pricing rows, formulas, edge cases, and source-of-truth paths.

## Edge Cases

- No telemetry samples produce `0.0` p50/p90 estimates with `no_data` confidence.
- Fewer than five matching telemetry samples fall back to global telemetry.
- Unknown token models use fallback pricing.
- Telemetry tracks cache and thinking token fields, but the current summary cost formula prices only input and output tokens.
- Cost rejection is strict: a model is rejected only when projected p90 tokens are greater than remaining tokens.
- Estimated or unknown quota data should not hard-deny admission.
- Generation budget caps reject a request only when existing spend plus estimated request cost exceeds the daily or monthly cap.

## Acceptance Criteria

1. `sdp/telemetry/cost-model.md` exists and uses the project telemetry Markdown style with clear sections for scope, source-of-truth paths, pricing, overhead, cap formulas, and edge cases.
   - **VERIFY:** `pytest tests/test_sdp_telemetry_cost_model.py -q`

2. The reference enumerates all current token pricing rows from the SDP cost summary helper, including fallback pricing.
   - **VERIFY:** `pytest tests/test_sdp_telemetry_cost_model.py::test_cost_model_documents_token_pricing -q`

3. The reference documents exact token, capacity, safe-concurrency, and generation budget formulas.
   - **VERIFY:** `pytest tests/test_sdp_telemetry_cost_model.py::test_cost_model_documents_cap_and_generation_formulas -q`

4. The reference documents fixed-overhead and edge-case behavior for reserve fraction, missing telemetry, unknown models, strict rejection, estimated quotas, and unpriced telemetry token fields.
   - **VERIFY:** `pytest tests/test_sdp_telemetry_cost_model.py::test_cost_model_documents_overhead_and_edge_cases -q`

5. Product-facing change notes and progress tracking mention the new telemetry cost-model reference.
   - **VERIFY:** `grep -n "cost-model.md" CHANGELOG.md progress.md`
