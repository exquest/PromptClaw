# Spec: T-018@20260416T054258Zb

## Problem Statement

The counterpoint rules registry (T-018@Za) added basic `verify_resolution()` and
`voice_leading_smoothness()` functions, but they operate on raw MIDI integers without
leveraging the rich interval consonance metadata in `music_theory.py`. A single
transition check cannot validate an entire voice-pair sequence, and smoothness scoring
ignores vertical interval quality entirely.

This task adds:
1. Dissonance metadata annotation using `music_theory.Interval` consonance data
2. Sequence-level resolution-completion verification (all sharp dissonances must
   resolve stepwise to a non-sharp-dissonance interval)
3. A resolution completeness score (0.0-1.0)
4. Consonance-weighted voice-leading smoothness scoring

## Technical Approach

All additions go into `my-claw/tools/senseweave/counterpoint_rules.py`. No new
modules. Imports only `Interval` and `interval_between` from `music_theory.py`
(already imported).

### New dataclasses

1. **`DissonanceAnnotation`** (frozen) — metadata for one vertical interval:
   - `beat_index: int` — position in the sequence
   - `leader_midi: int` — leader MIDI note at this beat
   - `follower_midi: int` — follower MIDI note at this beat
   - `interval: Interval` — the `music_theory.Interval` object for the vertical gap
   - `consonance: str` — extracted consonance level (perfect/imperfect/mild_dissonance/sharp_dissonance)
   - `resolved: bool | None` — True if dissonance resolved stepwise to non-sharp; None for consonances

2. **`DissonanceReport`** (frozen) — analysis of a complete voice-pair sequence:
   - `annotations: tuple[DissonanceAnnotation, ...]`
   - `sharp_dissonance_count: int`
   - `mild_dissonance_count: int`
   - `unresolved_count: int`
   - `resolution_rate: float` — ratio of resolved sharp dissonances to total (1.0 if none)

### New functions

3. **`analyze_dissonance(leader_notes: tuple[int, ...], follower_notes: tuple[int, ...]) -> DissonanceReport`**
   - Iterates paired notes, computes `interval_between` for each vertical pair
   - For each sharp dissonance at index `i`, checks resolution at `i+1`:
     - Both voices move stepwise (<=2 semitones)
     - The resulting vertical interval is NOT sharp_dissonance
   - Returns `DissonanceReport` with all annotations and summary counts

4. **`verify_resolution_sequence(leader_notes: tuple[int, ...], follower_notes: tuple[int, ...]) -> bool`**
   - Returns True only if every sharp dissonance in the sequence resolves properly
   - Delegates to `analyze_dissonance` internally

5. **`resolution_completeness_score(leader_notes: tuple[int, ...], follower_notes: tuple[int, ...]) -> float`**
   - Returns 0.0-1.0 measuring proportion of resolved sharp dissonances
   - 1.0 if no sharp dissonances or all resolved; 0.0 if none resolved

6. **`weighted_voice_leading_smoothness(leader_notes: tuple[int, ...], follower_notes: tuple[int, ...]) -> float`**
   - Computes base melodic motion average (like `voice_leading_smoothness`)
   - Adds a penalty of 2.0 semitones per unresolved sharp dissonance
   - A lower score still means smoother voice-leading

## Edge Cases

- Empty or single-note sequences: return clean report (no dissonances, rate 1.0)
- Mismatched sequence lengths: use `min(len(leader), len(follower))` pairs
- Sharp dissonance at last beat: cannot resolve (no next beat) — counts as unresolved
- All consonant intervals: report has zero counts, resolution_rate = 1.0
- Tritone (6 semitones) is sharp_dissonance per music_theory.py
- Minor/major second at beat boundary: minor second (1) is sharp, major second (2) is mild

## Acceptance Criteria

### AC-1: DissonanceAnnotation carries music_theory.Interval metadata
Each annotation in the report must reference the actual `Interval` object from
`music_theory.py`, not just a semitone count.

**VERIFY:** `pytest tests/test_counterpoint_rules.py::TestDissonanceMetadata::test_annotation_carries_interval_object -v`

### AC-2: Sharp dissonances are detected via consonance classification
`analyze_dissonance` must identify minor seconds (1 semitone), tritones (6), and
major sevenths (11) as sharp dissonances using `interval.consonance`.

**VERIFY:** `pytest tests/test_counterpoint_rules.py::TestDissonanceMetadata::test_sharp_dissonances_detected -v`

### AC-3: Resolution requires stepwise motion AND non-sharp target
A sharp dissonance is resolved only when both voices move <=2 semitones AND the
resulting interval is not itself a sharp dissonance.

**VERIFY:** `pytest tests/test_counterpoint_rules.py::TestDissonanceMetadata::test_resolution_requires_stepwise_and_consonant_target -v`

### AC-4: verify_resolution_sequence validates full sequences
Returns True only if all sharp dissonances in the sequence resolve properly.

**VERIFY:** `pytest tests/test_counterpoint_rules.py::TestDissonanceMetadata::test_verify_resolution_sequence_all_resolved -v`

### AC-5: resolution_completeness_score returns correct ratio
0.0 when no sharp dissonances resolve; 1.0 when all do or none exist.

**VERIFY:** `pytest tests/test_counterpoint_rules.py::TestDissonanceMetadata::test_resolution_completeness_score -v`

### AC-6: weighted_voice_leading_smoothness penalizes unresolved dissonances
The weighted version produces a higher (worse) score than the base version when
unresolved sharp dissonances are present.

**VERIFY:** `pytest tests/test_counterpoint_rules.py::TestDissonanceMetadata::test_weighted_smoothness_penalizes_unresolved -v`

### AC-7: Empty and single-note sequences handled gracefully
No errors, clean defaults (rate 1.0, smoothness 0.0).

**VERIFY:** `pytest tests/test_counterpoint_rules.py::TestDissonanceMetadata::test_empty_and_single_note -v`

### AC-8: Mismatched sequence lengths handled
