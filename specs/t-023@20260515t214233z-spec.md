# T-023@20260515T214233Z Specification

## Problem Statement

PAL deployment work now has a repo-managed manifest that names intended
host-managed `/opt/pal` files. The next deployment slice needs a deterministic
diff model that compares those local manifest entries against a remote
inventory without writing remote files, restarting services, backing up data, or
requiring live SSH in tests.

This task implements PAL-028 from
`sdp/prd-pal-2026-agentic-ops-platform.md`: report deploy diff sets. It must
remain local, typed, inspectable, and stdlib-only so later deploy-plan/apply
tasks can reuse it.

## Technical Approach

- Extend `promptclaw.pal_deploy` with deployment-state and diff dataclasses.
- Model a remote managed file snapshot with target path, content hash, mode,
  owner, group, and existence flag.
- Derive local managed file snapshots from a validated
  `PALDeploymentManifest` by hashing repo source files and carrying manifest
  metadata.
- Add a fake remote inventory helper for tests and later dry-run callers. It
  accepts target-path keyed content/metadata and produces remote snapshots
  without opening SSH.
- Add `diff_pal_deployment(...)` that returns deterministic diff sets:
  `added`, `changed`, `missing`, `unchanged`, and `unmanaged_remote`.
- Treat `added` as "managed manifest file absent remotely".
- Treat `changed` as "managed file exists remotely but content hash, mode,
  owner, or group differs".
- Treat `missing` as "managed file source is absent locally" for optional or
  unvalidated manifests; normal validated manifests should not produce this set.
- Treat `unmanaged_remote` as "remote inventory entry not declared by the
  manifest and not matched by manifest excluded-path patterns".
- Keep runtime/excluded paths such as logs, pid files, model storage, override
  flags, and `__pycache__` out of unmanaged warnings.
- Provide JSON-safe summaries for future CLI/artifact work.
- Do not add deploy-plan CLI, SSH reads, remote writes, backups, apply,
  rollback, service restarts, migrations, dependencies, or provider secrets.

## Edge Cases

- Diff output order must be deterministic by target path.
- Required source files are normally blocked by manifest validation; the diff
  model still reports missing local sources when invoked with an unvalidated or
  synthetic manifest.
- Metadata-only differences must count as `changed` even when content hashes
  match.
- Runtime files matched by excluded path globs must not appear as unmanaged
  remote files.
- Empty remote inventory must classify all present local manifest entries as
  `added`.
- Startup identity hardening remains an existing regression anchor. This PAL
  deploy diff task does not change startup flow because the current CLI,
  daemon, first-boot, and narrative ASGI tests already cover
  `bootstrap_identity()` persistence and ordering before `FirstBootAnnouncer`.
