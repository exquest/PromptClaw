# T-018: Wire generation status into operator_diagnostics.py

## Problem Statement
The face/inkplate display needs to show the current generation status (e.g., "♫ generating" or "♫ queued: N"). This information is written to `/tmp/generation_status.json` but is not currently read or exposed by `operator_diagnostics.py`.

## Technical Approach
1. Add a new function `generation_status(path: Path | None = None) -> dict[str, object]` to `my-claw/tools/senseweave/operator_diagnostics.py`.
2. The function will read `/tmp/generation_status.json` (or the provided path).
3. Handle missing/stale/corrupt files gracefully, defaulting to 0 queue depth and not generating.
4. Extract `queue_depth` as an integer.
5. Add a `caption` field to the returned dictionary:
   - "♫ queued: {N}" if `queue_depth > 1`
   - "♫ generating" if `queue_depth == 1` or `is_generating` is True
   - "♫ ready" otherwise

## Acceptance Criteria
- `operator_diagnostics.py::generation_status() -> dict` reads `/tmp/generation_status.json`
  - VERIFY: `pytest tests/test_operator_diagnostics.py -k test_generation_status_reads_file`
- Returns sanitized dict suitable for face/inkplate rendering
  - VERIFY: `pytest tests/test_operator_diagnostics.py -k test_generation_status_returns_dict`
- Adds a one-line caption: "♫ generating" when queue depth > 0, "♫ ready" otherwise
  - VERIFY: `pytest tests/test_operator_diagnostics.py -k test_generation_status_captions`
- Tests assert reader handles missing/stale/corrupt status file
  - VERIFY: `pytest tests/test_operator_diagnostics.py -k test_generation_status_handles_errors`
