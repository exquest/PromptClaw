# Spec: T-010@20260408T223256Zb

## Problem Statement

`ollama_health()` was added in T-010a but is not yet surfaced anywhere the operator can see it. The `/status` Telegram command shows only conversation count, running tasks, schedules, and artifacts — no Ollama instance health. The half-hour heartbeat payload similarly omits Ollama status. Operators have no way to see whether local LLM instances are alive without SSH.

## Technical Approach

1. **`/status` command** — Call `ollama_health()` inside `handle_builtin` for the `/status` branch. Pass the result into `CypherClawArt.status_display()` (add an optional `ollama` keyword parameter) so the display includes per-socket health, loaded models, and latency.

2. **Heartbeat payload** — Call `ollama_health()` inside `format_half_hour_heartbeat()` and add an `"ollama"` key to the returned payload dict. Also append a one-line Ollama summary to the heartbeat text.

3. **`CypherClawArt.status_display()`** — Extend with an optional `ollama: dict | None = None` parameter. When provided, render a compact Ollama section after the pet section showing per-socket status, model names, and latency.

4. **Integration test** — Test that `/status` output includes Ollama health text when `ollama_health()` returns healthy/unhealthy instances, and that the heartbeat payload contains the `"ollama"` key.

## Edge Cases

- If `ollama_health()` raises unexpectedly, `/status` and heartbeat must not crash — catch and render a fallback "Ollama: unavailable" line.
- If both sockets are unhealthy, the status should clearly indicate no local models are available.
- If models list is empty on a healthy socket, show "no models loaded".

## Acceptance Criteria

1. The `/status` Telegram command output includes per-socket Ollama health, loaded model names, and latency.
   VERIFY: `pytest tests/test_daemon_status_ollama.py::test_status_command_includes_ollama_health -q`

2. The half-hour heartbeat payload dict contains an `"ollama"` key with the `ollama_health()` result.
   VERIFY: `pytest tests/test_daemon_status_ollama.py::test_heartbeat_payload_includes_ollama -q`

3. The half-hour heartbeat text includes an Ollama summary line.
   VERIFY: `pytest tests/test_daemon_status_ollama.py::test_heartbeat_text_includes_ollama_line -q`

4. If `ollama_health()` raises, `/status` and heartbeat degrade gracefully instead of crashing.
   VERIFY: `pytest tests/test_daemon_status_ollama.py::test_status_degrades_on_ollama_error -q`

5. Existing `/status` and heartbeat tests continue to pass unchanged.
   VERIFY: `pytest tests/test_daemon_scheduler.py -q`
