# Specification: T-021@20260425T183959Zc

## Problem Statement

Sub-task `T-021@20260425T183959Za` added the combined sampler
plan+playback wording to the shared sample-status formatter, and
`T-021@20260425T183959Zb` wired that combined line into
`face_display.py`. The sampler PRD, however, calls for the same status
surface on the **inkplate/e-ink display** as well, and this checkout
does not currently contain any landed inkplate renderer module.

Without that seam, the shared combined sampler status never reaches an
e-ink surface, and there is no place to enforce the tighter width
constraints of a small monochrome display. Long combined status strings
can easily overflow such a surface.

## Technical Approach

1. Add a small `my-claw/tools/inkplate_display.py` renderer module rather
   than broadening unrelated display code. The renderer will be
   hardware-free and testable in-process.
2. Reuse the existing shared sampler wording by calling
   `senseweave.sample_status.face_display_sample_status_text(...)`.
   This keeps the face and inkplate surfaces on the same combined
   plan+playback contract instead of forking a second formatter.
3. Make truncation width-aware, not character-count-only: use Pillow
   text measurement (`ImageDraw.textbbox`) against the renderer's
   available content width and shorten with an ASCII `...` suffix until
   the line fits.
4. Keep the renderer seam small and composable: it should be able to
   build text lines from an optional `SurfaceSnapshot` plus the optional
   sampler status line, then render those lines onto a monochrome PIL
   image without any device I/O.
5. Keep the width configurable through the renderer constructor so tests
   can force narrow widths deterministically and future host-specific
   tuning does not require changing truncation logic.

## Edge Cases

- If the combined sampler helper returns an empty string, the renderer
  must omit the sample-status line entirely rather than emitting blank
  space or placeholder text.
- If the line already fits the available width, it must be preserved
  exactly and must not gain an ellipsis.
- If monitor state reports `no_capture`, that `monitor offline` wording
  should still flow through the shared helper and be truncated only when
  necessary for width.
- If the content width is narrower than the full combined line, the
  returned line must still fit within the measured width and end with
  `...`.
- The task is limited to the new renderer seam and its tests; it does
  not add device drivers, daemon startup hooks, or unrelated face-display
  changes.

## Assumptions

- No new dependencies or migrations are required; Pillow is already used
  elsewhere in the display stack.
- Because there is no landed inkplate renderer in this tree, creating a
  small renderer module is the narrowest way to satisfy the task without
  inventing broader display architecture.
- The generated hardening items about self-critique integration, budget
  enforcement, and startup identity sequencing are out of scope for this
  display-formatting task and will be treated as verification anchors.
- `progress.md` is generated and already dirty with unrelated queue-state
  updates, so only the single task line will be edited manually.

## Acceptance Criteria

1. The inkplate renderer uses the shared combined sampler-status helper
   and passes through the activity, playback, and monitor states.
   VERIFY: `pytest tests/test_inkplate_display.py::test_inkplate_renderer_uses_combined_sample_status_helper -q`

2. Long sampler status lines are truncated with `...` so the rendered
   text fits the configured e-ink content width.
   VERIFY: `pytest tests/test_inkplate_display.py::test_inkplate_renderer_truncates_long_sample_status_to_fit_width -q`

3. Sampler status lines that already fit the configured width are left
   unchanged.
   VERIFY: `pytest tests/test_inkplate_display.py::test_inkplate_renderer_keeps_sample_status_when_it_fits -q`

4. Empty or missing sampler state degrades cleanly by omitting the
   sample-status line instead of crashing or emitting filler text.
   VERIFY: `pytest tests/test_inkplate_display.py::test_inkplate_renderer_omits_empty_sample_status -q`
