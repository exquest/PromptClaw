# Task T-015@20260502T001347Z: PATCH /world/entities/{entity_id}

## Problem Statement

Deniable needs to apply resolved narrative consequences back into the shared
CypherClaw world state. The integration spec defines a `StateMutation` shape:
`target_entity`, `operation`, `field`, and `value`, with operations
`set`, `increment`, `decrement`, `append`, and `remove`.

The narrative API already supports listing, fetching, and creating world
entities, but it has no mutation endpoint. As a result, a Deniable fighter can
be created and read but cannot have gameplay state such as
`current_state="wounded"` persisted through the HTTP bridge.

## Technical Approach

- Extend `src/cypherclaw/narrative_api` rather than creating a new package.
- Add Pydantic schemas in `schemas.py`:
  - `StateMutation` with optional `target_entity` for path-compatible calls,
    required `operation`, required non-empty `field`, and `value`.
  - `EntityMutationRequest` with a non-empty `mutations` list.
  - The route accepts either a wrapped request (`{"mutations": [...]}`) or a
    single `StateMutation` object for integration-spec compatibility.
- Add an `apply_world_entity_mutations(...)` helper in `entities.py`:
  - Fetch the current entity through the existing `get_world_entity(...)`
    normalization path.
  - Treat bare field paths such as `current_state` as
    `properties.current_state`; explicit `properties.current_state` is also
    accepted.
  - Reject top-level immutable/control fields such as `id`, `type`, `name`,
    and `domain`; this endpoint mutates the entity `properties` payload only.
  - Apply operations in request order against a copy of `properties`.
  - Persist the updated entity through duck-typed store methods:
    `update_entity`, `patch_entity`, `save_entity`, `upsert_entity`, or
    `record_entity`.
  - Normalize the store return value when present and fall back to the
    computed updated record when the store returns `None`.
- Add `PATCH /world/entities/{entity_id}` in `app.py`:
  - Reuse shared-secret auth.
  - Map missing stores to HTTP `503`.
  - Map missing target entities to HTTP `404`.
  - Map validation/mutation errors to HTTP `422`.
  - Map downstream persistence failures to HTTP `502`.
  - Return the updated `EntityRecord`.
- Do not add dependencies or migrations. This mutates existing JSON payloads
  only.

## Edge Cases

- `target_entity` may be omitted because the path already identifies the
  target. When present, it must match `{entity_id}`.
- Empty mutation lists are rejected before the store is touched.
- Empty field paths, paths with empty segments, and attempts to mutate
  top-level `id`/`type`/`name`/`domain` are rejected.
- `set` creates missing nested dictionaries as needed.
- `increment` and `decrement` require numeric mutation values and numeric
  existing values. Missing fields start from zero.
- `append` creates a missing list, appends one value, and rejects non-list
  existing fields.
- `remove` removes matching values from lists. For non-list fields it removes
  the addressed property if present, making missing fields a no-op.
- All operations are applied to a copy; failed requests must not partially
  mutate the persisted store.
- The generated hardening bullets about self-critique, resource-governor
  budget enforcement, and daemon identity startup are unrelated to this HTTP
  wrapper. Existing tests remain verification anchors only.

## Acceptance Criteria

1. Mutating a Deniable fighter's `current_state` from `healthy` to `wounded`
   through `PATCH /world/entities/{entity_id}` persists and round-trips via
   `GET /world/entities/{entity_id}`.
   - **VERIFY:**
     `pytest tests/test_narrative_api_entities.py::TestPatchEntityEndpoint::test_set_current_state_persists_and_round_trips -q`

2. The endpoint applies `increment`, `decrement`, `append`, and `remove` on
   nested entity property paths in request order.
   - **VERIFY:**
     `pytest tests/test_narrative_api_entities.py::TestPatchEntityEndpoint::test_applies_numeric_list_and_nested_mutations_in_order -q`

   compatibility, not only the wrapped `{"mutations": [...]}` request shape.
   - **VERIFY:**
     `pytest tests/test_narrative_api_entities.py::TestPatchEntityEndpoint::test_accepts_single_state_mutation_payload -q`

   non-numeric numeric operations are rejected with HTTP `422` before any
