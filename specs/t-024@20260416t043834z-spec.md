# Specification: T-024@20260416T043834Z

## Problem Statement

CypherClaw's SenseWeave subsystem has a basic `derive_artistic_identity()` that
extracts top-2 families, patches, and image themes from song history, but it
lacks a comprehensive portfolio reporting layer that:

1. Incorporates **repertoire metrics** (total songs, promoted count, form class
   distribution, composition mode preferences, motif counts).
2. Incorporates **practice history** (which practice blocks have been visited,
   which EMSD course codes are covered).
3. Incorporates **growth state** (keys explored, ear quality averages from
   continuous learner data).
4. Provides **surface-ready data** for the face display and operator terminal —
   specifically current song title, section caption, practice block name, and
   an artistic-intent line derived from the identity.

The face display (`face_display.py`) already reads state from `/tmp/*.json` files.
A new reporting module should produce data structures that can be serialized to
the same pattern, but the module itself should remain pure (no I/O, no state
files, stdlib-only) — the caller writes the JSON.

## Technical Approach

### New module: `my-claw/tools/senseweave/portfolio_report.py`

A pure-function reporting module (no I/O, stdlib-only) that consumes repertoire
song data, growth state, and current-song context to produce two frozen
dataclasses.

### Data Model

```python
@dataclass(frozen=True)
class PortfolioReport:
    """Full artistic identity and repertoire portfolio summary."""
    # Core identity (reuses existing derivation)
    identity: ArtisticIdentity

    # Repertoire metrics
    total_songs: int
    total_promoted: int
    preferred_forms: tuple[str, ...]          # top form_class values
    preferred_modes: tuple[str, ...]          # top composition_mode values
    signature_motif_count: int                # unique motif IDs across score trees

    # Practice coverage
    practice_blocks_visited: tuple[str, ...]  # practice block names used
    course_codes_covered: tuple[str, ...]     # EMSD course codes touched

    # Growth metrics (from continuous learner state)
    keys_explored: tuple[str, ...]
    families_explored: tuple[str, ...]

    # Ear quality averages
    avg_hook_clarity: float
    avg_cadence_strength: float
    avg_development_score: float


@dataclass(frozen=True)
class SurfaceSnapshot:
    """Data for face/operator display surfaces."""
    song_title: str          # current song title, or "" if idle
    section_caption: str     # current section function, or "" if none
    practice_block: str      # current practice block name, or "" if none
    artistic_intent: str     # one-line artistic intent from identity
```

### Functions

```python
def derive_portfolio_report(
    songs: Sequence[Mapping[str, object]],
    *,
    promoted_songs: Sequence[Mapping[str, object]] = (),
    growth: Mapping[str, object] | None = None,
) -> PortfolioReport:
    """Build a portfolio report from repertoire and growth data."""

def derive_surface_snapshot(
    *,
    current_song: Mapping[str, object] | None = None,
    identity: ArtisticIdentity | None = None,
    practice_block: str = "",
    section_function: str = "",
) -> SurfaceSnapshot:
    """Build face/operator display snapshot from current song context."""
```

### derive_portfolio_report logic

1. Call `derive_artistic_identity(songs)` for the core identity.
2. Count `total_songs = len(songs)`.
3. Count `total_promoted = len(promoted_songs)`.
4. Extract top-2 `form_class` values from songs (Counter, most_common(2)).
5. Extract top-2 `composition_mode` values from songs (Counter, most_common(2)).
6. Count unique `motif_ids` across all `score_tree_summary` entries.
7. Collect unique `practice_block` values from songs.
8. Map practice block names to course codes via the `_BLOCKS` table in
   `practice_curriculum.py`.
9. Extract `keys_explored` and `families_explored` from either growth state
   or songs directly.
10. Compute average `hook_clarity`, `cadence_strength`, `development_score`
    from `ear_metrics` across all songs.

### derive_surface_snapshot logic

1. Extract `song_title` from current_song dict (default "").
2. Use `section_function` directly as caption (default "").
3. Use `practice_block` directly (default "").
4. Generate `artistic_intent` from identity statement if available,
   else empty string.

### Graceful degradation for missing data

- **Empty songs list**: Returns sensible defaults — `("bloom",)` for
  families, `("house_chamber",)` for patches, `("signal",)` for images
  (matching existing `derive_artistic_identity` behavior), zeros for metrics,
  empty tuples for collections.
- **Songs missing fields**: Each field extraction uses `.get()` with safe
  defaults. Missing `ear_metrics` yields 0.0 averages. Missing
  `score_tree_summary` contributes no motifs. Missing `practice_block`
  is skipped.
- **No growth state**: Growth-derived fields (`keys_explored`,
  `families_explored`) fall back to extracting from songs directly.
- **No current song**: Surface snapshot shows empty strings for all fields.
- **No identity**: Surface snapshot shows empty `artistic_intent`.

## Edge Cases

- **All songs have same family/form/mode**: Top-2 lists contain one element.
- **Songs with empty string fields**: Filtered out before counting.
- **Division by zero on averages**: When no songs have ear_metrics, averages
  are 0.0.
- **Corrupt score_tree_summary**: Missing `motif_ids` key treated as empty
  list.
- **Growth state with sets serialized as lists**: `keys_explored` in growth
  JSON is a list, not a set — handle both.
