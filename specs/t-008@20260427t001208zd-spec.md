# Problem Statement
The generation queue needs to be wired into the generation pipeline entry point. This involves updating `GenerationQueue` to accept its dependencies (`client`, `cache`, `budget`, `storage`) and adding a small integration test that enqueues a job, runs the worker loop once (end-to-end), and asserts the row reaches `done`.

# Technical Approach
- Update `GenerationQueue.__init__` to accept `client: Any | None = None`, `cache: Any | None = None`, `budget: Any | None = None`, `storage: Any | None = None`, `max_concurrent: int = 1`.
- Implement `process_one()` (or `run_worker(run_once=True)`) on `GenerationQueue` that:
  1. Calls `dequeue()`. Returns early if `None`.
  2. If `client` is present, calls `client.generate(item.payload)`.
  3. If `storage` is present, calls `storage.save(result, item.payload)`.
  4. Calls `mark_done(item.id)` on success.
  5. Catches exceptions and calls `mark_failed(item.id)`.
- Add an integration test `test_generation_queue_integration_worker_loop_success` using a fake client and fake storage that enqueues a request, runs `process_one()`, and verifies the row reaches `done`.

# Edge Cases
- Dependencies like `cache` and `budget` do not exist yet, so they must be optional (`None` by default) and ignored if absent.
- `process_one` handles transient failures by calling `mark_failed()`.

# Acceptance Criteria
1. `GenerationQueue` constructor accepts pipeline dependencies (`client`, `storage`, etc).
   - **VERIFY:** `pytest tests/test_generation_queue.py -k "constructor" -q`
2. `process_one` successfully processes a queued item, generates audio, stores it, and marks it done.
   - **VERIFY:** `pytest tests/test_generation_queue.py::test_integration_worker_loop_success -q`
