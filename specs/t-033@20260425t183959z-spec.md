# Specification: T-033@20260425T183959Z

## Problem Statement

CypherClaw now has a repo-backed `SampleRecord`, `SampleLibrary`, and
`SampleSelector`, but it still lacks a practical way to ingest Anthony's
existing recordings into the canonical sample-library layout. The current
store can add records one at a time, yet there is no manifest-driven bulk
import path, no deterministic copy into `samples/library/`, and no persisted
provenance field for the original source path or the human who captured the
recording.

That leaves CCS-032 incomplete in two ways:

1. There is no repeatable tool for migrating curated source audio into the
   library.
2. The library cannot preserve the provenance metadata required to justify
   future sampler choices.

## Technical Approach

Follow the existing `my-claw/tools/*.py` operator-script pattern and build on
the current `senseweave.sample_library` seam rather than adding a parallel
catalog.

### 1. Extend `SampleRecord` / `SampleLibrary` for provenance

- Add `extras` to `SampleRecord` for arbitrary JSON-compatible provenance
  metadata.
- Persist the full serialized `SampleRecord` in the library SQLite row so new
  metadata round-trips without having to add one DB column per future field.
- Keep the existing query columns (`source`, `character_tags`, `pitch_hz`,
  `arc_phase`, `mood`, `captured_at`) for filter performance.
- Perform the schema change through a backward-compatible inline SQLite upgrade
  in `SampleLibrary._ensure_schema()` because this store is a local per-library
  database with no separate migration framework in the tree.

### 2. Add a manifest-driven importer

- Create `my-claw/tools/sample_library_import.py`.
- Use stdlib `argparse` + `tomllib`.
- Read a TOML manifest with top-level defaults and `[[imports]]` entries.
- Support both:
  - single-file imports
  - directory imports with `recursive`, `include`, and `exclude` glob filters
- Copy source audio into `samples/library/` (or the caller-supplied
  `--library-root`) using deterministic, collision-safe destination names.
- Generate stable sample IDs from the original absolute path so rerunning the
  importer replaces the same logical record instead of endlessly duplicating
  it.
- Stamp `SampleRecord.extras` with at least:
  - `original_path`
  - `captured_by`
  - any manifest-supplied extra metadata

### 3. Commit a real manifest for Anthony's library migration

- Track a manifest in the repo for the real migration.
- Use the clearly available user-owned source corpus discovered during
  exploration:
  - 96 classifiable audio files from
    `/Users/anthony/Documents/from worker drive/Song Stems`
  - 5 handpicked single-file recordings from `/Users/anthony/Downloads`
- Split the directory import into non-overlapping filename buckets so tags are
  assigned by stem role instead of one vague tag set for the entire directory:
  - drums/percussion
  - bass/deep
  - chords/pads/groups/fx returns
  - melody/lead
  - vox/voice
  - theramin
- Add the 5 single-file recordings as explicit entries to satisfy the
  single-file import requirement in the real migration itself.

### 4. Keep generated audio out of git

- The manifest and code are tracked.
- The copied library audio and SQLite index are runtime artifacts under
  `samples/`; update `.gitignore` if needed so the migration does not pollute
  the repo with 100+ large binaries.

## Edge Cases

- Manifest paths may be absolute or relative; relative paths resolve from the
  manifest's parent directory.
- Directory imports ignore non-audio files.
- Source filenames may contain spaces and mixed case extensions.
- Rerunning the same manifest should be repeatable and should not create
  unbounded duplicate logical records for the same original path.
- Existing libraries created before this task may lack the new serialized-record
  column; opening them must self-upgrade in place.
- Missing source files should fail fast with a clear exception instead of
  silently skipping.

## Assumptions

- The canonical migration target for this task is the repo-local `samples/`
  root so imported files land under `samples/library/` and the index lives at
  `samples/index.sqlite`.
- No dedicated field-recordings directory was discoverable in Anthony's home,
  so the 5 explicit single-file recordings selected from `Downloads/` are the
  handpicked recording subset for this migration pass.
- No new third-party dependencies are required.
- The auto-generated startup-hardening checks are unrelated to this sampler
  import task and remain covered by the existing first-boot test surface.

   sample library.
   VERIFY: `pytest tests/test_sample_library_import.py::test_import_manifest_supports_single_file_and_directory_entries -q`

2. Imported records preserve provenance in `SampleRecord.extras`, including the
   original path and `captured_by`.
   VERIFY: `pytest tests/test_sample_library_import.py::test_imported_records_preserve_provenance_extras -q`

3. The importer CLI creates the library layout, copies audio into
   `samples/library/`, and reports the imported count.
   VERIFY: `pytest tests/test_sample_library_import.py::test_main_imports_manifest_and_reports_count -q`

   non-empty tags.
   VERIFY: `python my-claw/tools/sample_library_import.py sample_manifests/anthony-library.toml --library-root samples && python - <<'PY'\nimport json, sqlite3\nfrom pathlib import Path\ncon = sqlite3.connect('samples/index.sqlite')\ncount = con.execute('SELECT COUNT(*) FROM samples').fetchone()[0]\nassert count >= 101, count\nrows = con.execute('SELECT record_json FROM samples LIMIT 5').fetchall()\nassert rows, 'no imported rows'\nfor (payload,) in rows:\n    data = json.loads(payload)\n    assert data['character_tags'], data\n    assert data['extras']['original_path'], data\n    assert data['extras']['captured_by'], data\nprint(count)\nPY`
