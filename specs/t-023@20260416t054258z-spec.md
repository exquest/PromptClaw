# Task T-023 Specification: Course-driven self-critique and revision targets

## Problem Statement

The existing self-critique pipeline (`self_critique.py`) uses fixed, generic quality
thresholds (`DEFAULT_THRESHOLDS`) for all scores regardless of musical context. It
can detect that a score is "underdeveloped" or "static" but cannot name *which*
production-course concept failed — e.g. masking, lack of recurrence, weak cadence,
missing transition preparation, over-compression, or unsupported density. When a
revision attempt is made, it adjusts only the variation seed; it cannot steer the
revision toward specific course-concept improvements in form, harmony, rhythm,
register, mix, or strategy.

This task adds:

1. **Course-aware critique**: Map production-course chapter concepts to concrete
   ear-engine metrics so that critique output names the failing course concept, not
   just the raw metric.
2. **Course-driven thresholds**: Generate per-section quality thresholds from the
   section's `production_course` metadata so that each section is evaluated against
   what it *should* be doing musically.
3. **Bounded revision targets**: When revision is triggered, specify which musical
   dimensions (form, harmony, rhythm, register, mix, strategy) are open for
   adjustment — without changing piece identity (key, mood, tempo).
4. **Pipeline integration**: Wire course-aware critique into both the
   `compile_score_tree_to_tracker` and the `duet_composer` fallback path so every
   piece goes through course-driven critique before finalization.

## Technical Approach

### 1. Course-to-metric mapping (`self_critique.py`)

Add a `COURSE_METRIC_MAP` dict that maps `(chapter_id, value)` pairs to threshold
overrides:

- `("mode_scale", "static")` → `{"static_score": ("max", 0.4)}`
- `("harmonic_function", "dominant")` → `{"cadence_strength": ("min", 0.35)}`
- `("meter_groove", "groove")` → `{"onset_density": ("min", 1.5)}`
- `("counterpoint_relation", "imitative_canon")` → `{"repetition_score": ("min", 0.3)}`
- `("mix_role", "lead")` → `{"hook_clarity": ("min", 0.3)}`
- `("spatial_intent", "wide")` → `{"register_spread": ("min", 8.0)}` (via pitch_range_semitones)
- `("genre_strategy", "ambient_drone")` → `{"static_score": ("max", 0.85), "onset_density": ("max", 2.0)}`
- `("transition_type", "modulation")` → `{"cadence_strength": ("min", 0.25)}`

### 2. Course-aware threshold builder (`self_critique.py`)

New function `thresholds_for_course(course: dict[str, str]) -> dict[str, tuple[str, float]]`:
- Start from `DEFAULT_THRESHOLDS`.
- For each `(chapter_id, value)` in the section's `production_course`, look up
  `COURSE_METRIC_MAP` and merge (stricter threshold wins for same metric).
- Return the merged thresholds dict.

### 3. Course-aware CritiqueReport (`self_critique.py`)

New dataclass `CourseCritiqueReport` extending `CritiqueReport` with:
- `course_failures: tuple[CourseFailure, ...]` — each names a `(chapter_id, value,
  metric_name, direction, threshold, actual)`.

New function `critique_score_with_course(score, course)`:
- Calls `preview_score`, then `thresholds_for_course`, then `_check_thresholds`.
- For each failed metric, looks up which course concept contributed that threshold.
- Returns `CourseCritiqueReport`.

### 4. Revision targets (`self_critique.py`)

New frozen dataclass `RevisionTargets`:
- `adjust_form: bool`
- `adjust_harmony: bool`
- `adjust_rhythm: bool`
- `adjust_register: bool`
- `adjust_mix: bool`
- `adjust_strategy: bool`

New function `revision_targets_from_failures(course_failures)`:
- Maps each failed course concept to the dimensions it allows adjusting.
- E.g. `harmonic_function` failure → `adjust_harmony=True`.
- E.g. `meter_groove` failure → `adjust_rhythm=True`.
- E.g. `mix_role` failure → `adjust_mix=True`.

### 5. Extended `revise_score` (`self_critique.py`)

Add optional `course: dict[str, str] | None` parameter to `revise_score()`.
When provided:
- Use `thresholds_for_course(course)` instead of `DEFAULT_THRESHOLDS`.
- Build `CourseCritiqueReport` for the original score.
- Compute `RevisionTargets` from failures.
- Attach `course_failures` and `revision_targets` to the `RevisionResult`.

Extended `RevisionResult` with:
- `course_failures: tuple[CourseFailure, ...] = ()`
- `revision_targets: RevisionTargets | None = None`

### 6. Pipeline wiring

**`tracker_compiler.py`**: `compile_score_tree_to_tracker` already has access to
`section_courses` (line 1395). Pass the first section's `production_course` as the
`course` kwarg to `revise_score()`.

**`duet_composer.py`**: The fallback path (line 838) calls `revise_score` without
course context. Add `course` kwarg derived from the score tree's first section if
available.

### 7. Integration test

Add a test that verifies:
- A piece with known course metadata goes through course-aware critique.
- Failed metrics map to named course concepts.
- After revision, at least one metric improves.
- The final score is selected from the two-pass result.

## Edge Cases

- **No production_course metadata**: Falls back to `DEFAULT_THRESHOLDS` (existing behavior).
- **Unknown course concept value**: Skipped in threshold merge — only known mappings apply.
- **Conflicting thresholds from multiple chapters**: Stricter threshold wins per metric.
- **All metrics pass on first attempt**: No revision triggered, `course_failures` is empty.
- **Feature flags disabled**: Same early-return behavior as existing `revise_score`.
- **Empty score or minimal phrases**: `preview_score` handles gracefully with fallback metrics.

## Assumptions

- The ear-engine metrics (`_analyze_events` + `feedback_scores`) are stable and sufficient
  for course-concept evaluation without adding new metrics.
- `CriticalListeningReport.masking_risk` is available via `_critical_listening()` but is
  not yet in the `preview_score` path — if needed for masking critique, it will be added
  to the preview metrics dict.
- Budget enforcement (grain_density, density_multiplier) and bootstrap_identity are
  already wired in the current codebase — confirmed during exploration.
