# Specification: T-025@20260416T043834Z

## Problem Statement

CypherClaw now has score trees, section functions, arrangement automation,
ear-analysis feedback, EMSD sampling, a live master bus, and self-listening
state, but the operator `/status` surface still only shows daemon counters,
pets, and Ollama. When the installation sounds wrong, the operator cannot see
which musical tree is active, which section function is performing, which
arrangement curve is driving density/master automation, whether ear metrics are
available, which sample source is actually sounding, whether the master bus is
alive, or whether the self-listener is reporting real audio.

The recent SenseWeave rollout also needs coarse controls so curriculum/exercise
selection, preview render, self-critique/revision, and long-form suite behavior
can be enabled or disabled independently during live deployment and rollback.

## Technical Approach

1. Add a small, stdlib-only `senseweave.operator_diagnostics` module that reads
   the existing authority files and returns a typed diagnostic snapshot:
   - `/tmp/current_score_tree.json`
   - `/tmp/tracker_runtime_state.json`
   - `/tmp/composer_state.json`
   - `/tmp/sample_dsp_activity.json`
   - `/tmp/sample_playback_state.json`
   - `/tmp/master_bus_state.json`
   - `/tmp/self_listen.json`
2. Include score-tree title/id/form, current section function, arrangement curve
   or automation summary, compact ear metrics, sample source plan/execution,
   master bus health/values, self-listener RMS/backend/error, and feature flag
   states in the diagnostic snapshot.
3. Extend `cypherclaw_daemon.status_snapshot()` and
   `CypherClawArt.status_display()` so Telegram `/status` includes a compact
   SenseWeave diagnostic block while degrading cleanly if state files are
   missing or corrupt.
4. Add a `senseweave.rollout_controls` module with independent boolean flags:
   - `CYPHERCLAW_ENABLE_CURRICULUM_EXERCISE`
   - `CYPHERCLAW_ENABLE_PREVIEW_RENDER`
   - `CYPHERCLAW_ENABLE_SELF_CRITIQUE`
   - `CYPHERCLAW_ENABLE_LONG_FORM_SUITE`
5. Wire the flags into the relevant behavior without changing current defaults:
   - curriculum/exercise disabled -> away-practice selection falls back to
     `performance_weave`.
   - preview-render disabled -> `revise_score()` skips offline metrics.
   - self-critique disabled -> `revise_score()` returns the original score
     without attempting a rewrite.
   - long-form suite disabled -> `commission_piece()` downshifts `suite` to
     `extended` while preserving existing non-suite form decisions.
6. Document safe rollout order and rollback files in the operator docs and
   changelog.

## Edge Cases

- Missing or corrupt state files must not crash `/status`; diagnostics should
  render explicit `unknown` or `unavailable` values.
- A stale or missing master-bus state file may still have current composer
  master-bus values. The diagnostic should show both values and health when
  possible.
- Playback state should override sample plan when it claims a sample is
  currently playing, but requested and resolved sample sources must both remain
  visible when they differ.
- Self-critique cannot run when preview rendering is disabled; the raw flags
  stay independent, but effective revision behavior requires both preview and
  self-critique enabled.
- The default flag state preserves existing behavior so this task does not
  silently roll back already-live SenseWeave features.
- Startup identity hardening is not modified by this task, but remains a
  required verification anchor.

## Assumptions

- No new external dependencies are required; all changes use the Python standard
  library.
- The operator status surface for this task is Telegram `/status`, backed by
  `cypherclaw_daemon.status_snapshot()` and `CypherClawArt.status_display()`.
- Rollback for these controls is file/env driven: unset or set the relevant
  `CYPHERCLAW_ENABLE_*` variable to `0`, restore the listed changed Python/docs
  files from git, and restart the minimal affected process.
