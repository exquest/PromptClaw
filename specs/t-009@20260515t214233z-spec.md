# T-009@20260515T214233Z Spec: Add Slow-Inference Context Collection

## Problem Statement

PAL-018 needs a bounded workflow that captures the evidence operators need when
PAL 2026 inference is slow: router health, baseline token throughput, GPU
resource hints, and relevant logs when those remote read-only diagnostics are
available. The existing PAL triage/action workflows can gather broad health
signals, but there is no dedicated slow-inference context artifact that a later
diagnosis command can consume without re-running ad hoc checks.

## Technical Approach

- Add a `run_pal_slow_inference_context(...)` workflow function in
  `promptclaw/pal_agent.py` beside the existing triage/action workflows.
- Preserve the current local-control-plane model: no arbitrary shell commands,
  no infrastructure mutation, no provider secrets in source, and remote checks
  only through fixed read-only SSH helpers.
- Use the standard `.promptclaw/runs/<run-id>/` layout with `ArtifactManager`,
  `RunState`, events, route metadata, a JSON output artifact, a handoff, and a
  short local summary.
- Reuse `pal_health` and smoke-baseline summary behavior for health and saved
  baseline data.
- Add fixed read-only tools for slow-inference context:
  - `gpu_hints`: query `nvidia-smi` summary/process hints over configured SSH.
  - `slow_inference_logs`: tail bounded PAL router/Ollama/startup logs over
    configured SSH.
- Derive a top-level baseline token/s value from saved smoke reports when
  prompt-level token/s data exists, while preserving the full baseline summary.
- Skip optional SSH-backed GPU/log tools safely when `PAL_SSH_HOST`,
  `PAL_SSH_PORT`, and `PAL_SSH_KEY` are not all set.
- Do not add the operator CLI yet; PAL-019 owns the slow-inference diagnosis CLI.
- Do not add dependencies, migrations, database columns, or mutating actions.

## Edge Cases

- No smoke reports exist: baseline status is `skipped`, token/s aggregate is
  `None`, and the workflow still completes.
- Smoke reports contain no token/s values: preserve the baseline summary and set
  aggregate token/s to `None`.
- SSH diagnostics are not configured: GPU and log tools report `skipped` with
  the existing safe skip command shape.
- Remote GPU/log commands return non-zero: capture stdout/stderr and mark the
  observation `warn` instead of failing the workflow.
- A tool raises or returns non-object data: reuse `_run_tool(...)` error wrapping
  so the workflow writes an artifact rather than crashing.
- Startup hardening bullets target existing identity startup code; this task
  re-runs those regression anchors and does not broaden unrelated startup flow.
