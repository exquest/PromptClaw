# Spec: Operator Diagnostics for Production-Course Decisions

**Task:** T-025@20260416T054258Z
**Tier:** T2 — Standard

## Problem Statement

The operator `/status` display shows generic SenseWeave state (score tree, section
function, arrangement curve, ear metrics, sample source, master bus, self-listener)
but does not expose the production-course decisions that drive composition behavior.
Operators cannot see which phase, scale/mode, harmonic function, groove, production
strategy, mix role, spatial profile, counterpoint/Theramini relation, or critique
notes are active for the current section without inspecting raw JSON files.

## Technical Approach

### Data Sources

Production-course decisions live in two authority files:

1. **`/tmp/tracker_runtime_state.json`** — `scene_metadata` carries `production_*`
   prefixed keys (e.g., `production_mode_scale`, `production_phase_profile`) written
   by `tracker_compiler.py` line 1266-1269.

2. **`/tmp/current_score_tree.json`** — each section's `production_course` dict
   carries the same chapter IDs. Used as fallback when tracker runtime is stale.

3. **`/tmp/composer_state.json`** — `arc_phase` in extras provides phase when
   tracker metadata is absent.

4. **`/tmp/theramini_state.json`** — Theramini playing state, pitch, note name.
   Combined with `production_counterpoint_relation` for Theramini relation display.

5. **Critique notes** — not currently persisted to an authority file. The diagnostics
   will show `unavailable` when absent. If `ear_metrics` contain failed thresholds,
   a compact note can be derived from the metric values themselves.

### Changes

#### `operator_diagnostics.py`

- Add `theramini` path to `DiagnosticPaths`.
- Add three new fields to `OperatorDiagnostics`:
  - `production_course: str` — compact one-line summary of active chapter values
    (phase, scale/mode, harmonic function, groove, strategy, mix role, spatial).
  - `theramini_relation: str` — counterpoint relation + Theramini playing state.
  - `critique_notes: str` — compact critique summary or "unavailable".
- Add `_production_course_status()` — reads `production_*` keys from scene_metadata,
  falls back to score_tree current section, then composer_state arc_phase.
- Add `_theramini_relation_status()` — reads counterpoint_relation + theramini state.
- Add `_critique_notes_status()` — derives notes from ear_metrics thresholds if
  available, otherwise "unavailable".
- Update `to_status_dict()` to include all new fields.

#### `glyphweave/scenes.py`

- Update `status_display()` to render new fields in the SenseWeave section:
  - `Production course: phase=Emergence scale=modal harmonic=tonic ...`
  - `Theramini: contrary_motion (playing C4)` or `Theramini: unavailable`
  - `Critique: unavailable` or `Critique: hook_clarity below 0.2`

### Edge Cases

- All production_* keys missing from scene_metadata → fall back to score_tree
  sections, then show "unavailable".
- Score tree has no sections → "unavailable".
- Theramini state file missing or corrupt → "unavailable".
- Theramini not playing but counterpoint_relation present → show relation only.
- Ear metrics missing → critique notes show "unavailable".
- Only some production-course keys present → show available ones, omit absent ones.

- Production-course keys in scene_metadata use the `production_` prefix convention
  established by `tracker_compiler.py`.

  a new authority file.
