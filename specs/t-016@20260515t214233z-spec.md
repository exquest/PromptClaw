# T-016@20260515T214233Z Specification

## Problem Statement

PAL operators need one read-only command that audits the Phase 1 auto-shutdown
setup. Existing PAL commands can validate router health and propose approval
gated shutdown override actions, but there is no reproducible workflow artifact
that states whether scheduled shutdown is enabled, whether the override flag is
active, and when the next shutdown window will occur.

This task implements the PAL-021 `shutdown_audit` workflow from
`sdp/prd-pal-2026-agentic-ops-platform.md`.

## Technical Approach

- Add a deterministic local workflow in `promptclaw/pal_agent.py` named
  `run_pal_shutdown_audit(...)`.
- Expose the workflow as:

  ```bash
  promptclaw pal audit shutdown PROJECT_ROOT
  ```

- The workflow uses fixed read-only SSH diagnostics only:
  - Read `/opt/pal/config/shutdown.conf` values for `ENABLED`,
    `SHUTDOWN_TIME`, `TIMEZONE`, and `OVERRIDE_FILE`.
  - Check whether the configured override flag is present.
  - Inspect crontab for `/opt/pal/scripts/auto_shutdown.sh`.
  - Capture current local time in the configured shutdown timezone.
  - Tail recent `/opt/pal/logs/shutdown.log` lines.
- PromptClaw derives a local JSON-safe audit result with:
  - `shutdown_enabled_state`: `enabled`, `disabled`, or `unknown`.
  - `override_state`: `active`, `inactive`, or `unknown`.
  - `next_shutdown_window`: a five-minute local-time window, or
    `unavailable` when config/time evidence is incomplete.
  - cron/config/log evidence and `mutating_actions: []`.
- Write standard run artifacts under `.promptclaw/runs/<run-id>/`:
  - `outputs/shutdown-audit.json`
  - `routing/route.json`
  - `routing/route.md`
  - `handoffs/shutdown-audit.md`
  - `summary/final-summary.md`
  - `logs/events.jsonl`
  - `state.json`
- Treat the command as read-only except for local PromptClaw artifacts.

## Edge Cases

- Missing PAL SSH environment variables record the remote diagnostic as
  `skipped`; the summary still states `unknown` shutdown enabled state,
  `unknown` override state, and `unavailable` next shutdown window.
- Missing or unreadable `shutdown.conf` records config as unavailable and does
  not infer an enabled or disabled state.
- Invalid `SHUTDOWN_TIME`, missing current local time, or malformed time strings
  produce `next_shutdown_window: unavailable`.
- Missing cron entry records `cron_installed: false` and `audit_status: warn`.
- An active override flag records `override_state: active` and
  `audit_status: warn` because the next scheduled shutdown would be skipped.
- Disabled shutdown records `shutdown_enabled_state: disabled` and
  `audit_status: warn` without changing the remote config.
- The workflow never touches `/opt/pal/config/override.flag`, edits cron,
  shuts down the host, exposes approval action ids, or calls the PAL router.
- The generated startup hardening bullets remain regression anchors for the
  existing identity startup subsystem; this PAL audit workflow does not alter
  startup identity behavior.
