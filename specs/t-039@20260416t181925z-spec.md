# Specification: T-039@20260416T181925Z

## Problem Statement

CCH-034 currently gives the render gate symbolic novelty evidence through score
fixtures and proxy metrics, but it does not validate the audio that a listener
would actually hear. A render can therefore pass symbolic checks while the
synthesis layer flattens dynamics, timbre, or section contrast enough that the
audio novelty disappears.

T-039 adds an offline, deterministic 5-minute proxy-audio path and a librosa
analysis pass that extracts a self-similarity matrix and Foote novelty curve from
the rendered WAV. The audio novelty peaks are then cross-checked against the
symbolic novelty peak times derived from the CCH-034 fixture metadata.

## Technical Approach

### New module: `my-claw/tools/senseweave/render/audio_novelty.py`

1. Define typed dataclasses:
   - `ProxyAudioRender`: rendered WAV path, sample rate, duration, and symbolic
     novelty peak times used during rendering.
   - `AudioNoveltyAnalysis`: square self-similarity matrix, Foote novelty curve,
     frame times, and detected audio novelty peak times.
   - `NoveltyAlignment`: symbolic peaks, detected audio peaks, per-peak matched
     audio time/deviation, maximum deviation, and pass/fail status.

2. Implement `symbolic_novelty_peak_times(part: PerformedPart) -> tuple[float, ...]`:
   - Prefer explicit `symbolic_novelty_peak_times_s` from CCH-034 fixture
     metadata.
   - Accept `novelty_peak_times_s` as an alias.
   - If only `novelty_peaks` exists, derive evenly spaced symbolic checkpoints
     across the fixture duration.
   - Clamp/validate peak times to the open interval `(0, duration)`.

3. Implement `render_proxy_audio(...) -> ProxyAudioRender`:
   - Render a mono PCM WAV using a deterministic oscillator/timbre proxy.
   - Use `duration_sec` from the fixture and default to 300 seconds.
   - Make segment boundaries coincide with symbolic novelty peak times so the
     generated audio contains audible timbre/pitch/dynamic transitions.
   - Provide a test-only `flatten_expression=True` path that intentionally
     ignores those segment changes while leaving symbolic metrics untouched.

4. Implement `extract_audio_novelty(...) -> AudioNoveltyAnalysis`:
   - Load the rendered WAV with `librosa.load`.
   - Extract mel features with `librosa.feature.melspectrogram` and
     `librosa.power_to_db`.
   - Build a normalized self-similarity matrix from those audio features.
   - Compute a Foote checkerboard novelty curve over the matrix diagonal.
   - Detect local novelty peaks from the normalized curve.

5. Implement `check_audio_symbolic_novelty_alignment(...) -> NoveltyAlignment`:
   - Match each symbolic peak to the nearest audio novelty peak.
   - Pass only when every symbolic peak has an audio peak within `tolerance_s`
     (default `0.5`, matching the ±500 ms requirement).
   - Return enough detail to explain flattened or misaligned render failures.

6. Export the new public API from `senseweave.render`.

### Dependency

Add `librosa>=0.10,<1` to project dependencies. It is required for the actual
audio-feature extraction path in this task.

## Startup Hardening Check

The auto-generated hardening item is not part of the audio-rendered novelty
feature, but it is mandatory for this task. The current tree already calls
`bootstrap_identity()` before `FirstBootAnnouncer` in both daemon entrypoints.
Add a small persistence regression test that calls `bootstrap_identity()` twice
for standalone and federated modes and verifies the second boot reuses the
persisted identity record.

## Edge Cases

- Fixture has no symbolic novelty peaks: renderer still writes 5 minutes of
  proxy audio; alignment passes only if no symbolic peaks were requested.
- Fixture has peak times outside the piece duration: ignore invalid boundaries
  instead of writing impossible segment breaks.
- Flattened synthesis: symbolic metrics can stay healthy, but the audio novelty
  cross-check must fail because no audio peak aligns within 500 ms.
- Missing librosa: import failure should clearly state that librosa is required
  for audio novelty extraction.
- Very sparse audio: novelty curve normalizes safely without division by zero.

## Assumptions

- "Actual rendered audio" means the deterministic offline proxy WAV produced by
  this render layer, not live SuperCollider/JACK output.
- The CCH-034 symbolic novelty peak source is represented by fixture metadata
  (`symbolic_novelty_peak_times_s` or `novelty_peak_times_s`). Existing
  count-only fixtures can derive evenly spaced checkpoints, but alignment tests
  use explicit times.
- No database migrations are required because this feature writes no database
  schema or persistent state.
