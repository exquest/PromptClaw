# T-007@20260515T214233Z Specification

## Problem Statement

PAL-016 makes the local PAL knowledge base usable after PAL-015 writes
`.promptclaw/pal-kb/index.jsonl`. Operators need a deterministic command that
searches the local PAL KB and returns ranked snippets with source paths, without
calling the live PAL router or introducing a heavyweight search dependency.

The acceptance criterion is: "Query returns ranked snippets with source paths."

## Technical Approach

- Keep the implementation in `promptclaw/pal_knowledge.py` beside source
  discovery, chunking, and index writing.
- Add a frozen `PALKnowledgeQueryResult` dataclass with typed fields:
  `rank`, `score`, `chunk_id`, `source_path`, `start_line`, `end_line`, and
  `snippet`.
- Add `query_pal_knowledge_index(project_root, query, index_path=None,
  limit=5)`:
  - load the default `.promptclaw/pal-kb/index.jsonl` unless an override is
    provided;
  - parse one JSON object per line into `PALKnowledgeChunk` records;
  - tokenize the query and chunk text with a small stdlib regex tokenizer;
  - rank by deterministic lexical overlap with a small exact-phrase boost;
  - drop non-matching chunks;
  - sort by score descending, then source path, line range, and chunk id for
    stable ties;
  - return bounded snippets containing the best query term when possible.
- Add `promptclaw pal kb query PROJECT_ROOT --query TEXT [--limit N]
  [--index PATH] [--json]`.
- Print concise ranked human output by default and a JSON result list when
  `--json` is passed.
- Keep query local and stdlib-only; no router calls, database migrations,
  provider secrets, or new dependencies.

## Edge Cases

- A blank or whitespace-only query raises `ValueError`.
- A non-positive limit raises `ValueError`.
- A missing index raises `FileNotFoundError` with the requested path.
- Malformed JSONL rows raise `ValueError` identifying the bad line.
- Empty indexes return an empty result list and a human summary with
  `matches=0`.
- Query matching is case-insensitive.
- Snippets collapse whitespace and remain bounded so CLI output stays readable.
- The generated startup identity hardening bullets target the existing
  identity/startup subsystem. Current startup tests already cover
  `bootstrap_identity()` persistence and ordering before `FirstBootAnnouncer`;
  this task re-runs those anchors rather than changing unrelated startup flow.

## Acceptance Criteria

   line ranges, scores, chunk ids, and bounded snippets.
   VERIFY: `pytest tests/test_pal_knowledge.py::test_query_pal_knowledge_index_returns_ranked_snippets_with_source_paths -q`

   chunks.
   VERIFY: `pytest tests/test_pal_knowledge.py::test_query_pal_knowledge_index_is_case_insensitive_and_deterministic -q`

   VERIFY: `pytest tests/test_pal_knowledge.py::test_query_pal_knowledge_index_rejects_invalid_inputs -q`

   VERIFY: `pytest tests/test_pal_knowledge.py::test_pal_kb_query_cli_prints_ranked_snippets -q`
