# T-008@20260515T214233Z Spec: Inject PAL Knowledge Into Workflow Prompts

## Problem Statement

PAL workflow prompts currently ask the remote PAL router to plan and summarize
bounded operations using only the task text, tool/action descriptions, and live
diagnostic JSON. The local PAL knowledge base from PAL-013 through PAL-016 can
already discover, chunk, index, and query source files, but PAL workflow prompt
artifacts do not yet include those snippets. Operators therefore cannot verify
which local PAL runbook or deployment facts were available to PAL when a plan or
summary was generated.

## Technical Approach

- Keep PAL execution local-control-plane first: PromptClaw still writes prompt
  artifacts, calls PAL for reasoning, and executes only local allow-listed
  tools/actions.
- Add a bounded `Knowledge Context` prompt section built from
  `query_pal_knowledge_index(...)`.
- Query with the operator task text and use a small fixed result limit so prompt
  growth stays deterministic.
- Render source path, line range, score, chunk id, and bounded snippet text.
- Inject the same bounded context into PAL triage plan/summary prompts and PAL
  action plan/summary prompts.
- Missing, empty, or malformed local KB indexes must not fail the workflow; the
  prompt artifact should still include the `Knowledge Context` section with a
  short unavailable-context note.
- Do not contact PAL while building knowledge context beyond the existing
  workflow query calls.
- Do not add migrations, dependencies, remote writes, provider secrets, or new
  approval-gated actions.

## Edge Cases

- KB index has more matching rows than the prompt should carry: include only the
  fixed maximum number of matches.
- KB index is missing because the operator has not run `promptclaw pal kb build`:
  include a bounded unavailable-context note and continue.
- KB index exists but returns no lexical matches: include a bounded no-match note
  and continue.
- KB row snippets include long text: reuse bounded query snippets and clamp the
  rendered section.
- Workflow task text is empty or whitespace-only: do not raise from knowledge
  lookup; render the unavailable-context note.
