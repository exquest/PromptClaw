# T-006@20260515T214233Z Specification

## Problem Statement

PAL-015 finishes the first local PAL knowledge-base slice by materializing the
deterministic chunk stream from PAL-014 into an on-disk index. Operators need a
single command, `promptclaw pal kb build PROJECT_ROOT`, that writes a local KB
artifact without contacting the live PAL router. The required artifact is
`.promptclaw/pal-kb/index.jsonl` or a SQLite equivalent; this task chooses JSONL
to keep the writer inspectable, append-compatible with later prompt-injection
work, and stdlib-only.

The acceptance criterion is: "`pal kb build` creates
`.promptclaw/pal-kb/index.jsonl` or SQLite equivalent."

## Technical Approach

- Keep the implementation in `promptclaw/pal_knowledge.py` beside source
  discovery and chunking.
- Add a typed build result dataclass with the index path, source count, chunk
  count, and chunk size limit.
- Add `write_pal_knowledge_index(project_root, config=None, max_chars=2000,
  output_path=None)`:
  - load config once when not supplied;
  - use `discover_pal_source_files(...)` for source count and
    `chunk_pal_source_files(...)` for deterministic chunk payloads;
  - write JSON Lines, one `PALKnowledgeChunk` payload per line;
  - sort JSON object keys for stable byte output;
  - create `.promptclaw/pal-kb/` as needed;
  - write through a temporary sibling file and replace the destination so a
    failed write does not leave a partial index behind.
- Add a nested CLI path:
  `promptclaw pal kb build PROJECT_ROOT [--max-chars N] [--output PATH]
  [--json]`.
- Print a concise human summary by default and a JSON build summary when
  `--json` is passed.
- Keep the writer local and stdlib-only; no router calls, database migrations,
  provider secrets, or new dependencies.

## Edge Cases

- Empty/whitespace-only source files produce an existing but empty
  `index.jsonl`.
- Missing source files and broad directory matches inherit source-discovery
  ignore behavior.
- Invalid `max_chars` raises `ValueError` before creating/replacing the index.
- Re-running the writer with unchanged source files produces identical bytes.
- A custom output path is allowed for tests and operator diagnostics; the
  default path remains `.promptclaw/pal-kb/index.jsonl`.
- UTF-8 source text is preserved through JSON escaping rules.
- The generated startup identity hardening bullets target the existing
  identity/startup subsystem. Current startup tests already cover
  `bootstrap_identity()` persistence and ordering before `FirstBootAnnouncer`;
  this task re-runs those anchors rather than changing unrelated startup flow.

## Acceptance Criteria

1. `write_pal_knowledge_index(...)` creates the default JSONL index path with
   deterministic chunk payloads and a typed build summary.
   VERIFY: `pytest tests/test_pal_knowledge.py::test_write_pal_knowledge_index_creates_default_jsonl_index -q`

2. Rebuilding the index for unchanged source files is byte-stable and replaces
   stale content.
   VERIFY: `pytest tests/test_pal_knowledge.py::test_write_pal_knowledge_index_is_stable_and_replaces_stale_content -q`

3. `promptclaw pal kb build` writes the index and reports the build summary.
   VERIFY: `pytest tests/test_pal_knowledge.py::test_pal_kb_build_cli_writes_index_and_prints_summary -q`

4. Startup identity hardening anchors still pass.
   VERIFY: `pytest tests/test_cli_identity_hardening.py tests/test_first_boot.py::TestStartupIdentityPersistence tests/test_first_boot.py::TestStartupIdentityModePersistence tests/test_governor_integration.py::TestStartupIdentityWiring tests/test_narrative_api_main.py::test_asgi_module_startup_bootstraps_identity_persistence_between_imports -q`

5. Product-facing change notes mention the PAL KB index writer.
   VERIFY: `grep -n "PAL KB index writer" CHANGELOG.md progress.md`

6. Full validation passes before final commit.
