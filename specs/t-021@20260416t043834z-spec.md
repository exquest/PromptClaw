# Spec: T-021@20260416T043834Z

## Problem Statement

CypherClaw currently generates musical scores and plays them immediately. There is
no pre-performance quality check: a piece with poor interval variety, weak hook
clarity, or excessive repetition goes straight to the speakers. The continuous
learner reflects *during* and *after* playback, but by then the audience has already
heard the weak material.

This task adds a self-critique layer that runs *before* performance when practical.
A generated score is converted to a fast offline preview (frequency events), analyzed
by the ear engine, and gated against quality thresholds. If metrics fail, the system
requests one bounded rewrite pass with a different variation seed, then picks
whichever version scores better ‚Äî provided it improves at least one failed metric
without changing the piece's identity (key, mood, tempo, family).

## Technical Approach

1. **New module `my-claw/tools/senseweave/self_critique.py`** ‚Äî sits between
   `generative_scores.score_from_mood()` and playback. Stdlib only, no new deps.

2. **`preview_score(score: Score) -> dict[str, float]`** ‚Äî converts a Score to
   frequency events via `score_to_frequencies()`, builds `_EarEvent` objects from
   those frequencies, and runs `ear_engine._analyze_events()` to produce the full
   metric dict. This is the "offline fast preview render" required by AC-1.

3. **`critique_score(score: Score, thresholds: Mapping | None) -> CritiqueReport`**
   ‚Äî wraps `preview_score()` and compares each metric against configurable
   thresholds. Returns a frozen dataclass with `passed: bool`,
   `failed_metrics: tuple[str, ...]`, and the raw metrics.

   Default thresholds (all corrective scores are bad-high, quality scores are
   bad-low):
   - `underdeveloped_score` > 0.7 ‚Üí fail
   - `static_score` > 0.7 ‚Üí fail
   - `development_score` < 0.25 ‚Üí fail
   - `hook_clarity` < 0.2 ‚Üí fail

4. **`revise_score(mood, *, thresholds, max_passes=1, **mood_kwargs) -> RevisionResult`**
   ‚Äî generates an initial score via `score_from_mood()`, critiques it. If it
   passes, returns immediately. If it fails, regenerates with `variation_seed + 1`
   (one bounded rewrite pass, AC-2). Compares original and revised metrics, picks
   the version where at least one failed metric improved. Verifies identity
   preservation (same key, mood string, tempo_bpm).

5. **Dataclasses:**
   - `CritiqueReport(metrics, passed, failed_metrics)`
   - `RevisionResult(original_score, original_metrics, revised_score,
     revised_metrics, improved_metrics, identity_preserved, revision_used)`

## Edge Cases

- Score with very few notes (sleeping mode, 4 notes) ‚Äî thresholds may naturally
  fail for short pieces. `critique_score` applies to any score; callers decide
  whether to gate sleeping pieces.
- All metrics pass on first try ‚Äî `revise_score` returns the original with
  `revision_used=False`.
- Revision is worse on all failed metrics ‚Äî system keeps the original, returns
  `revision_used=False` and `improved_metrics=()`.
- `variation_seed` is None ‚Äî rewrite pass uses seed 1 for the original and seed 2
  for the revision.
- Custom thresholds override defaults ‚Äî caller can tighten or loosen any metric.
