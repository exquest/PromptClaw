## Problem Statement

CypherClaw's harmonic and melodic generation currently relies heavily on Western diatonic, modal, and symmetric scales. To expand the expressive range into avant-garde, ambient, and modern contemporary spaces, the system needs post-tonal, spectral, and microtonal material strategies. This allows the composer to generate pitch-class sets, harmony derived from the harmonic series (spectralism), just-intonation ratios, and 24-TET (quarter-tone) events. Additionally, SenseWeave's listening capabilities should include spectral analysis to extract prominent frequencies from the environment to seed this harmonic material.

## Technical Approach

1. **Extend `my-claw/tools/senseweave/music_theory.py`:**
   - **Post-tonal:** Add a `PitchClassSet` dataclass and functions to generate sets from prime forms (e.g., `[0, 1, 4]`).
   - **Spectral:** Add `spectral_partials(fundamental_hz, count, stretch=1.0)` to generate a list of frequencies representing the harmonic (or inharmonic) series.
   - **Microtonal (24-TET):** Add `quarter_tone_to_freq(midi_note_qt: float, a4=440.0)` where `midi_note_qt` allows half-integer values (e.g., 60.5) to compute 24-TET frequencies.
   - **Just Intonation:** Extend just intonation support with `just_intonation_chord(base_freq, ratios)` to return a list of frequencies.

2. **Extend `my-claw/tools/audio_analysis.py`:**
   - Add a pure Python `extract_spectral_peaks(samples, sample_rate, max_peaks=4)` that computes a simple FFT (or DFT) on a window of samples to find the strongest frequency bins, allowing SenseWeave to seed spectral harmony from live audio.

3. **Extend `my-claw/tools/senseweave/harmonic_planner.py`:**
   - Add support for generating a `HarmonicPlan` tagged with `post-tonal`, `spectral`, or `microtonal` strategies when requested by the composer phase/style.
   - The planner can use `PitchClassSet`, `spectral_partials`, or `quarter_tone_to_freq` to bound the generated material.

4. **Hardening (Auto-Generated):**
   - Ensure `bootstrap_identity()` is correctly invoked before `FirstBootAnnouncer` in `cypherclaw_daemon.py`.

## Edge Cases

- **Performance:** Pure Python FFT must be limited to a small window (e.g., 1024 or 2048 samples) to avoid blocking the audio analysis thread.
- **Out of bounds:** 24-TET and spectral frequencies must be clamped to safe audio ranges (e.g., 20 Hz to 12000 Hz) to avoid aliasing or inaudible generation.
- **Phase/Style matching:** The harmonic planner should only use these avant-garde strategies when the piece's phase or style permits (e.g., `intent="avant-garde"` or `phase="Development"`).

## Assumptions

- No new external dependencies required (stdlib only).
- Hardening checks around `bootstrap_identity` in `cypherclaw_daemon.py` must be addressed as mandated.
