# T-022@20260530T002730Z Spec: Asset Music Render CLI

## Problem Statement

The Deniable Asset Bus producer can safely pass renderer argv to the
CypherClaw box, and the image side now has a thin box-side entrypoint. The
music side still needs the matching command contract:
`tools/asset_render_music.py` must accept scene, mood, duration, loopability,
and output path arguments, then write one WAV file at that output path. The
contract must be testable in this repo without a live CypherClaw box,
SuperCollider server, GPU, or provider credentials.

## Technical Approach

- Add `tools/asset_render_music.py` as a thin box-side CLI wrapper.
- Parse the documented argv into a typed `MusicRenderParams` dataclass:
  required `--scene`, repeatable required `--mood`, required positive
  `--duration`, optional `--loopable`, and required `--output`.
- Keep parsing separate from rendering so tests can assert the exact contract
  without booting the synthesis stack.
- Render one WAV file to `params.output`, creating the parent directory if
  needed.
- Use a replaceable renderer callback for tests. The default renderer will be a
  deterministic standard-library fallback that writes a valid silent WAV; it
  documents where the deployed box can swap in the existing CypherClaw synthesis
  stack without changing the argv contract.
- Extend `FakeBoxRunner` narrowly so a test invocation of
  `asset_render_music ... --output <name>.wav` can materialize a WAV artifact
  under the supplied `output_dir`, preserving argv recording and the no-shell
  boundary.
- Do not add dependencies, provider secrets, database columns, migrations,
  startup-flow changes, or asset-bus wire-contract changes.

## Edge Cases

- `--duration` must parse as a positive float; zero, negative, NaN, infinity,
  and non-numeric values fail during argument parsing.
- `--mood` is repeatable and must be supplied at least once because the v0.1
  music spec carries a mood list.
- `--scene` must be non-empty after trimming whitespace.
- `--output` may point at a missing parent directory; the directory is created.
- `--loopable` is a bare flag and defaults to `False`.
- Fake-runner music output is confined under `output_dir`; absolute output
  paths outside that directory are rejected instead of written.
- The auto-generated startup identity hardening bullets are unrelated to this
  box-side renderer task. Existing integration coverage already verifies
  `bootstrap_identity()` invocation, ordering before `FirstBootAnnouncer`, and
  persistence across standalone and federated boots, so those tests are run as
  mandatory regression anchors rather than broadening the implementation.

## Acceptance Criteria

1. `tools/asset_render_music.py` exposes a typed parser for the documented argv
   and resolves `--scene`, repeatable `--mood`, `--duration`, `--loopable`, and
   `--output` into `MusicRenderParams`.
   VERIFY: `pytest tests/test_asset_render_music.py::test_parse_documented_argv_resolves_render_parameters -q`

2. The CLI render path writes one valid WAV file at the requested output path
   using an injected renderer in tests.
   VERIFY: `pytest tests/test_asset_render_music.py::test_render_music_writes_wav_with_injected_renderer -q`

3. Invalid duration values, missing mood values, and blank scenes fail during
   argument parsing before any renderer is called.
   VERIFY: `pytest tests/test_asset_render_music.py::test_parse_rejects_invalid_duration_mood_and_scene -q`

4. `FakeBoxRunner` reproduces the documented `asset_render_music` output
   contract in repo tests: it records argv verbatim and materializes the WAV
   named by `--output` under the provided `output_dir`.
   VERIFY: `pytest tests/test_asset_bus_runner.py::test_fake_box_runner_reproduces_asset_render_music_output_contract -q`

5. Required startup identity hardening anchors still pass: startup paths invoke
   `bootstrap_identity()`, identity is created before first-boot announcement,
   and identity persists between standalone and federated boots.
   VERIFY: `pytest tests/test_cli_identity_hardening.py tests/test_first_boot.py::TestStartupIdentityPersistence tests/test_first_boot.py::TestStartupIdentityModePersistence tests/test_midi_intake_daemon.py::test_main_invokes_bootstrap_identity tests/test_midi_intake_daemon.py::test_identity_persistence_between_boots tests/test_narrative_api_main.py::test_asgi_module_startup_bootstraps_identity_persistence_between_imports -q`

6. Product-facing task notes mention the new music render CLI, and no new
   dependencies or migrations are recorded.
   VERIFY: `rg -n "asset_render_music|T-022@20260530T002730Z" CHANGELOG.md progress.md ESCALATIONS.md`
