# Task T-001@20260502T001347Z: POST /memory/search

## Problem Statement

Deniable needs to query CypherClaw's shared narrative memory over HTTP instead
of importing the narrative engine or opening its SQLite state directly. The
PromptClaw queue has imported the narrative HTTP service PRD, but this checkout
does not contain the deploy-side `cypherclaw.narrative` package yet. The first
slice is `POST /memory/search`: accept `{query, k, domain_filter}`, call
`NarrativeMemory` semantic search, and return ranked JSON results.

## Technical Approach

- Add a small `cypherclaw.narrative_api` package under `src/`, following the
  adjacent `cypherclaw.image_api` pattern: Pydantic schemas, a `create_app(...)`
  factory, and dependency injection for tests.
- Implement `POST /memory/search` with an injected memory searcher when provided
  and a lazy deploy-time import of `cypherclaw.narrative.memory.NarrativeMemory`
  otherwise.
- Prefer a `semantic_search(query, k, domain_filter)` method when available,
  but tolerate nearby search method names (`search`, `query`) and common
  parameter names (`k`, `top_k`, `limit`) so the wrapper remains thin across
  small deploy-side signature differences.
- Normalize common result shapes into a stable wire schema:
  `{id, text, similarity, domain, metadata}`.
- Sort results by `similarity` descending in the API layer before returning.
- Pass `domain_filter` through to the memory engine and defensively post-filter
  returned results when they expose a known `domain`.
- Return `503` when no default `NarrativeMemory` can be imported and `502` when
  the underlying memory engine raises during a request.
- Do not add database columns, migrations, orchestration changes, systemd units,
  or new dependencies in this slice.

## Edge Cases

- Empty or whitespace-only `query` is invalid and should produce a Pydantic
  validation error.
- `k` must be positive and bounded so callers cannot ask the service for an
  unbounded result set.
- `domain_filter` may be a single domain string, a list of domains, or omitted.
- If the memory engine returns a known domain not present in `domain_filter`, the
  API excludes that hit even if the engine failed to filter it.
- If the memory engine returns tuple, dict, dataclass, or object-shaped hits, the
  wrapper extracts best-effort text, score, domain, and metadata without leaking
  unserializable objects.
- Results with missing scores normalize to `0.0`, which keeps the endpoint
  stable while making lower-confidence results sort behind scored hits.
- Generated candidate hardening items for `bootstrap_identity()` are not part of
  this endpoint; the existing startup identity tests remain the regression
  anchors.
