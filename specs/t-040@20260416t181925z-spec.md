# Specification: T-040@20260416T181925Z

## Problem Statement

Rendered pieces carry rule-stack provenance (via iXML and JSON sidecars from
`render.export`), and the ablation debugger (CCH-010, `senseweave-render-debugger`
CLI) can localize expressive problems to individual rules. However, there is no
operator-facing workflow that ties these capabilities together: no documented
cadence for reviewing weekly output, no structured log for recording listener
impressions, and no operator guide explaining how to ablate a suspect rule and
attach the result to a review entry.

T-040 closes this gap with three deliverables: an operator guide document, a
weekly review log template, and tests proving the artifacts exist and correctly
reference the ablation CLI.

## Technical Approach

### 1. Operator guide: `my-claw/docs/listener-review.md`

Markdown document covering:

- **Cadence**: weekly review of all rendered pieces from the past 7 days.
- **How to ablate a rule**: step-by-step walkthrough using the
  `senseweave-render-debugger` CLI from CCH-010. Include the CLI invocation
  pattern with `--score`, `--renderer`, `--rules`, and `--format` flags.
  Reference the JSON sidecar (`.meta.json`) as the source for the rule stack
  of a given piece.
- **How to attach a review note**: instructions for adding an entry to the
  weekly review log (`my-claw/sdp/review-log.md`), including the required
  fields and an example entry.

### 2. Weekly review log: `my-claw/sdp/review-log.md`

Markdown file with:

- Header explaining the log purpose.
- Template entry with fields: `piece` (filename or ID), `date` (review date),
  `felt_wrong` (free-text symptom description), `suspected_rule` (rule ID from
  provenance), `ablation_result` (summary of debugger output or "N/A"),
  `action` (keep / disable / tune / escalate).
- One example entry filled in to demonstrate the format.

### 3. Validation module: `my-claw/tools/senseweave/render/listener_review.py`

Minimal Python module providing:

- `LISTENER_REVIEW_DOC_PATH`: resolved `Path` to `my-claw/docs/listener-review.md`.
- `REVIEW_LOG_PATH`: resolved `Path` to `my-claw/sdp/review-log.md`.
- `ABLATION_CLI_NAME`: string constant `"senseweave-render-debugger"` (the
  CCH-010 CLI prog name from `debugger.build_parser().prog`).
- `validate_listener_review_artifacts() -> bool`: checks that both files exist
  and that the doc references the ablation CLI name.

This module gives tests a stable import target and keeps path resolution in one
place.

### Dependency

No new external dependencies.

## Edge Cases

- Review log is empty (no entries yet): template header and example entry must
  still be present so the file is parseable.
- Operator guide references CLI flags that change: the doc references the
  `build_parser().prog` constant rather than hardcoding, and the test validates
  the match.
- Paths resolve correctly from both the repo root and the tools directory.
