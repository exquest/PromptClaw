# Specification: T-023@20260416T043834Z

## Problem Statement

CypherClaw's SenseWeave subsystem currently has rich sensor fusion (presence
engine, cadence engine, world model) and detailed DSP activity planning
(sample_dsp_activity, sample_lab, dsp_scene_lab), but no unified **acoustic
ecology policy** layer that translates installation context into concrete
output constraints.  The cadence engine sets target parameters, but nothing
enforces hard ceilings or encodes the ecological principle that environmental/
keynote sounds should sometimes be privileged over generated material.

A living-space sound installation must respect the acoustic ecology of the
space: sleep must be truly quiet, wind-down must never spike, occupied-active
should lean into the room's own sonic character, and away-practice can open
up for experimentation.

## Technical Approach

### New module: `my-claw/tools/senseweave/acoustic_ecology.py`

A pure-function policy module (no I/O, no state files, stdlib-only) that
consumes:

- `occupancy_state` (from presence engine: occupied_active, occupied_quiet,
  likely_asleep, likely_away, uncertain)
- `cadence_state` (from cadence engine: sleep, wake_ramp, wind_down,
  away_practice, occupied_day)
- `day_phase` (from cadence engine: late_night, pre_dawn, morning_activation,
  mid_morning, midday, afternoon_dip, late_afternoon, evening_settling,
  pre_sleep)
- `room_activity` (quiet, moderate, active)
- `attention_state` (ambient, attending, performance)
- `dwell_time_s` (float, seconds since reliable presence began)
- `hour` (int, 0-23)

And produces an `AcousticEcologyPolicy` frozen dataclass:

```python
@dataclass(frozen=True)
class AcousticEcologyPolicy:
    # Ecology mode label
    ecology_mode: str  # sleep, wind_down, quiet_occupied, active_day,
                       # away_practice, performance

    # Hard ceilings (must never be exceeded)
    max_loudness_db: float
    max_onset_density: float      # events per second
    max_spectral_centroid_hz: float  # brightness ceiling

    # Silence windows
    silence_probability: float    # 0.0-1.0
    min_silence_gap_s: float
    max_silence_gap_s: float

    # Source preferences
    preferred_sources: tuple[str, ...]  # ordered preference
    keynote_privilege: float      # 0.0-1.0, weight for env/keynote sounds
    generated_material_weight: float  # 0.0-1.0

    # Density/voice constraints
    max_voices: int
    density_ceiling: float        # 0.0-1.0

    # Contextual metadata
    reasons: tuple[str, ...]
```

### Ecology mode resolution

The `resolve_ecology_mode()` function maps the combination of inputs to one
of six ecology modes:

| Priority | Condition | Mode |
|----------|-----------|------|
| 1 | cadence=sleep OR occupancy=likely_asleep | sleep |
| 2 | cadence=wind_down | wind_down |
| 3 | attention=performance | performance |
| 4 | cadence=away_practice OR occupancy=likely_away | away_practice |
| 5 | occupancy in (occupied_active) AND room_activity in (moderate,active) | active_day |
| 6 | fallback | quiet_occupied |

### Policy tables

Each ecology mode defines a base policy.  Modifiers then adjust based on:

- **Dwell time**: Short dwell (<120s) softens transitions (10% lower ceilings
  than steady state).  Long dwell (>1800s) can relax slightly (+5%) as the
  occupant is settled.
- **Room activity**: Active rooms increase `keynote_privilege` (the room IS
  the soundscape) and decrease `generated_material_weight`.
- **Day phase**: Late night / pre_sleep phases tighten brightness ceiling
  even within non-sleep modes.

### Sleep and wind-down ceilings (hard enforcement)

These are the non-negotiable ceilings that tests will verify:

| Parameter | Sleep Ceiling | Wind-down Ceiling |
|-----------|--------------|-------------------|
| max_loudness_db | 40.0 | 48.0 |
| max_onset_density | 0.5 | 1.5 |
| max_spectral_centroid_hz | 800.0 | 1200.0 |
| density_ceiling | 0.15 | 0.35 |
| max_voices | 2 | 3 |

### Environmental/keynote privilege

When `keynote_privilege > 0.5`, environmental sources (room_mic, garden_mic,
contact_mic) should be preferred over generated material.  The
`preferred_sources` tuple orders sources by ecological priority:

- **sleep**: `("room_mic", "garden_mic")` — ambient keynotes only
- **wind_down**: `("room_mic", "garden_mic", "contact_mic")` — gentle env
- **quiet_occupied**: `("room_mic", "contact_mic", "theramini_in", "self_bus")`
- **active_day**: `("room_mic", "contact_mic", "theramini_in", "self_bus", "garden_mic")`
- **away_practice**: `("self_bus", "room_mic", "contact_mic", "theramini_in", "garden_mic")`
- **performance**: `("theramini_in", "room_mic", "contact_mic", "self_bus")`

## Edge Cases

- **Stale presence (uncertain)**: Fail safe to quiet_occupied with moderate
  ceilings, not away_practice.
- **Transition from sleep to wake_ramp**: cadence_state=wake_ramp maps to
  quiet_occupied with dwell-time modifier (the occupant just woke up).
- **Performance during wind-down hours**: performance mode overrides wind-down
  ceilings because the human is actively engaged (but still caps brightness
  below full active_day levels).
- **No dwell_time provided (0.0)**: Treat as fresh arrival with transition
  softening.
