# T-004@20260515T214233Z Specification

## Problem Statement

PAL-013 starts the PAL knowledge-base slice by adding a deterministic source
discovery function. Later PAL chunking and index-writing tasks need one local
function that reads PromptClaw project configuration and returns the configured
PAL knowledge input files without contacting the live PAL router or mutating
runtime artifacts.

The acceptance criterion is: "Test returns configured sample files."

## Technical Approach

- Extend `PALConfig` with a `knowledge_sources` list of glob or file patterns.
- Add a small PAL knowledge/source module under `promptclaw/`.
- Implement `discover_pal_source_files(project_root, config=None)`:
  - load `promptclaw.json` when no config is provided;
  - expand configured `pal.knowledge_sources` entries relative to
    `project_root`;
  - allow absolute patterns for future operator-supplied source lists;
  - return only existing files;
  - de-duplicate repeated matches by resolved path;
  - sort output deterministically by resolved POSIX path.
- Keep the function stdlib-only and side-effect free.
- Update the committed `pal-2026/promptclaw.json` to declare the current PAL
  source set used by future KB tasks.

## Edge Cases

- Missing configured files or empty glob matches are ignored.
- Directories matched by a broad glob are ignored.
- Duplicate matches from overlapping patterns appear once.
- An empty configuration falls back to the default PAL knowledge source
  patterns so existing PAL configs continue to work.
- The function does not parse or read file contents; unreadable file content is
  handled by later chunk/index tasks.
- Startup identity hardening is outside this source-discovery change, but the
  existing regression anchors must be re-run because the task prompt carries the
  recurring bootstrap checks.

## Acceptance Criteria

1. Configured sample PAL source files are discovered deterministically.
   VERIFY: `pytest tests/test_pal_knowledge.py -q`

2. PAL configuration round-trips the new knowledge source list.
   VERIFY: `pytest tests/test_pal_client.py::PALConfigTests::test_pal_config_round_trips_through_project_config -q`

3. Default PAL source discovery remains local and side-effect free for existing
   configs.
   VERIFY: `pytest tests/test_pal_knowledge.py::test_discover_pal_source_files_uses_default_pal_sources_when_config_is_empty -q`
