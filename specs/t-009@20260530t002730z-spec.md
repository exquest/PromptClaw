# T-009@20260530T002730Z Spec: SSHBoxRunner Argv-Safe Remote Transport

## Problem Statement

The Deniable Asset Bus now has a `BoxRunner` protocol and an in-process
`FakeBoxRunner`, but production rendering still needs a real transport to the
CypherClaw box. That transport must preserve the asset request argv contract:
prompts, scenes, moods, and other request-derived values are data, not shell
syntax. An injection-shaped request string must reach the renderer verbatim as
one argv element and must never be interpolated into a local or remote shell
command line.

## Technical Approach

- Add `SSHBoxRunner` in `promptclaw/asset_bus/runner.py` and export it from
  `promptclaw.asset_bus`.
- Keep the existing `BoxRunner.run(argv, *, output_dir)` shape so future
  producer code can swap `FakeBoxRunner` and `SSHBoxRunner` without changing
  routing code.
- Invoke local `ssh` through `subprocess.run([...], shell=False)` with a fixed
  argv list containing only transport configuration and the fixed remote Python
  module entrypoint.
- Send the request argv to the remote entrypoint as JSON stdin instead of
  placing request-derived strings in the SSH command argv.
- Add a small remote helper module that reads JSON stdin, validates it as a
  list of strings, creates the remote output directory, and runs the child
  renderer as an argv list with `shell=False`.
- Transfer generated files back with an argv-list `rsync` command from the
  remote run directory to the local `output_dir`, then return the local files
  in `BoxRunResult.artifacts`.
- Keep tests fully local by injecting a fake subprocess runner; no unit test may
  open a real network connection or require a live CypherClaw host.
- Do not add dependencies, provider secrets, database columns, migrations, or
  startup-flow changes.

## Edge Cases

- Request-derived values include shell metacharacters such as `;`, `&&`,
  `$()`, backticks, pipes, redirects, spaces, or newlines: preserve the value as
  one JSON argv element and one remote child argv element.
- `argv` is empty, contains a non-string value, or contains a NUL byte: reject
  before invoking SSH.
- Remote render command exits nonzero: preserve stdout/stderr and return the
  renderer exit status; output transfer remains best-effort for partial files.
- Remote render succeeds but transfer fails: return the transfer exit status
  and include transfer stderr.
- Local output directory does not exist: create it before transfer.
- Generated startup-identity hardening bullets are out of scope for this
  runner transport slice; verify existing CLI, daemon, standalone/federated,
  and ASGI identity persistence anchors remain green.

## Acceptance Criteria

1. `SSHBoxRunner` constructs both SSH and transfer invocations as argv lists,
   passes `shell=False`, and never places request-derived strings in the SSH or
   transfer command arguments.
   VERIFY: `pytest tests/test_asset_bus_runner.py::test_ssh_box_runner_invokes_ssh_and_rsync_as_argv_lists -q`

2. An injection-shaped fixture is serialized verbatim as one argv element for
   the remote argv helper and is not split or shell-expanded.
   VERIFY: `pytest tests/test_asset_bus_runner.py::test_ssh_box_runner_serializes_injection_fixture_as_verbatim_argv_element -q`

3. The remote argv helper validates JSON input and invokes the renderer with an
   argv list and `shell=False`, appending only the fixed output-directory
   option.
   VERIFY: `pytest tests/test_asset_bus_remote_exec.py -q`

4. Asset-bus runner and render-arg regression tests pass without network access.
   VERIFY: `pytest tests/test_asset_bus_runner.py tests/test_asset_bus_remote_exec.py tests/test_asset_bus_render_args.py -q`

5. Product-facing docs and progress mention the SSHBoxRunner transport boundary
   and its argv/no-shell contract.
   VERIFY: `grep -n "SSHBoxRunner" docs/architecture.md docs/handoff-protocol.md docs/command-reference.md docs/startup-wizard.md CHANGELOG.md progress.md`
