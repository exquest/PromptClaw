# Spec: T-012@20260408T223256Z

## Problem Statement

PromptClaw instances running on the same Tailscale network have no way to discover each other. Federation requires peer awareness before any message exchange, task delegation, or knowledge sharing can occur. The first step is a discovery mechanism that scans the local tailnet for other PromptClaw instances by probing each Tailscale peer for the `/federation/v1/identity` endpoint, then stores discovered peers in a local registry.

## Technical Approach

1. **Tailscale status parsing** — Run `tailscale status --json` to enumerate all peers on the tailnet. Extract each peer's DNS name and IP addresses. Handle the case where Tailscale is not installed or not connected.

2. **Identity probing** — For each discovered Tailscale peer, probe `http://{peer_ip}:{port}/federation/v1/identity` (default port 8443) with a short timeout. If the endpoint returns a valid `InstanceIdentity` JSON payload, the peer is a PromptClaw instance.

3. **Peer registry** — Store discovered peers in `~/.promptclaw/federation/peers.json` with:
   - `instance_id`, `instance_name`, `mode`, `capabilities` (from identity response)
   - `tailscale_ip`, `tailscale_name` (from tailscale status)
   - `endpoint` (the probed URL)
   - `first_seen`, `last_seen` timestamps
   - `online` boolean

4. **Merge semantics** — On re-scan, update existing peers (bump `last_seen`, update `online` status) rather than replacing the file. Peers that were previously seen but are now offline get `online: false` with their last known data preserved.

5. **Module location** — `promptclaw/federation/discovery.py` with supporting `promptclaw/federation/__init__.py`. Follows the planned federation package layout from `prd-federation.md`.

6. **No new external dependencies** — Uses `subprocess` for `tailscale status --json` and `urllib.request` for HTTP probes (stdlib only). No `httpx`, `requests`, or `cryptography` needed for this task.

## Edge Cases

- Tailscale not installed → return empty peer list, log warning.
- Tailscale installed but not connected (no peers) → return empty peer list.
- Peer is on tailnet but not running PromptClaw → probe returns non-200 or non-JSON → skip.
- Peer is on tailnet, running PromptClaw, but identity endpoint is malformed → skip with warning.
- `~/.promptclaw/federation/` directory does not exist → create it.
- `peers.json` exists with stale data → merge, marking unreachable peers as offline.
- Network timeout on probe → treat as offline, do not block scan.
- Multiple IPs per peer → try each until one succeeds or all fail.
- Self-discovery → exclude the local machine's Tailscale IP from probing.

## Acceptance Criteria

1. `discover_peers()` returns a list of `PeerInfo` dataclasses for each PromptClaw instance found on the tailnet.
   VERIFY: `pytest tests/test_federation_discovery.py::test_discover_peers_finds_promptclaw_instances -q`

2. When Tailscale is not installed or not connected, `discover_peers()` returns an empty list without raising.
   VERIFY: `pytest tests/test_federation_discovery.py::test_discover_peers_tailscale_unavailable -q`

3. Peers that do not expose the identity endpoint are excluded from results.
   VERIFY: `pytest tests/test_federation_discovery.py::test_discover_peers_skips_non_promptclaw_peers -q`

4. `save_peer_registry()` persists peers to `~/.promptclaw/federation/peers.json`.
   VERIFY: `pytest tests/test_federation_discovery.py::test_save_peer_registry_creates_file -q`

5. `load_peer_registry()` reads back the persisted registry.
   VERIFY: `pytest tests/test_federation_discovery.py::test_load_peer_registry_reads_saved_data -q`

6. Re-scanning merges with existing registry: updates `last_seen` for known peers, marks unreachable peers as offline, adds new peers.
   VERIFY: `pytest tests/test_federation_discovery.py::test_merge_peer_registry_updates_existing_and_marks_offline -q`

7. The local machine's own Tailscale IP is excluded from probing.
   VERIFY: `pytest tests/test_federation_discovery.py::test_discover_peers_excludes_self -q`

8. Malformed identity responses are skipped gracefully.
   VERIFY: `pytest tests/test_federation_discovery.py::test_discover_peers_handles_malformed_identity -q`
