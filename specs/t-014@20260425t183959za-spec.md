# Specification: T-014@20260425T183959Za

## Problem Statement

`CCS-020` in the sampler PRD requires a persistent systemd unit for
`sample_capture_daemon.py`, but the current tree has no
`my-claw/systemd/cypherclaw-sample-capture.service` file and no boot-script
hook that starts it on the live host.

That leaves the sampler capture daemon as a manual process instead of a
boot-resilient service, even though the surrounding CypherClaw runtime already
manages other long-lived components through repo-backed systemd units.

## Technical Approach

1. Add `my-claw/systemd/cypherclaw-sample-capture.service` following the
   existing CypherClaw unit conventions:
   - `User=user`
   - `WorkingDirectory=/home/user/cypherclaw`
   - `ExecStart=/home/user/cypherclaw/.venv/bin/python3 /home/user/cypherclaw/tools/sample_capture_daemon.py`
   - `Restart=always`
   - `After=cypherclaw-jack.service`
   - `Requires=cypherclaw-jack.service`
2. Preserve the PRD-specific runtime requirement for JACK-backed audio capture
   by setting `LimitMEMLOCK=infinity` in the unit.
3. Extend `my-claw/scripts/cypherclaw_boot.sh` with a small helper that
   best-effort starts `cypherclaw-sample-capture.service`, matching the script's
   existing pattern for repo-owned or repo-referenced auxiliary services.
4. Add focused regression tests that pin both:
   - the unit-file contract
   - the boot-script service-start hook

## Edge Cases

- The repository references `cypherclaw-jack.service` but does not ship that
  unit. This task should treat it as an external prerequisite and only wire the
  dependency, not invent a new JACK service definition.
- `sample_capture_daemon.py` uses the SuperCollider self bus in addition to JACK
  capture ports. The service should still depend only on `cypherclaw-jack.service`
  per the PRD, and rely on `Restart=always` plus the boot-script start hook to
  recover cleanly if the full audio stack comes up slightly later.
- The boot hook should be best-effort, not fatal, so a transient `systemctl`
  failure does not abort the rest of `cypherclaw_boot.sh`.
- The work stays scoped to service wiring and task-tracking artifacts; no
  runtime behavior inside `sample_capture_daemon.py` changes in this task.

## Assumptions

- The correct live runtime user remains `user`, matching the existing CypherClaw
  service files already in `my-claw/systemd/`.
- The correct interpreter path remains
  `/home/user/cypherclaw/.venv/bin/python3`, matching the PRD and existing
  repo-backed services.
- No new dependencies or migrations are required.

## Acceptance Criteria

1. The repository ships a sample-capture systemd unit with the required user,
   venv, restart, and JACK dependency wiring.
   VERIFY: `pytest tests/test_cypherclaw_boot_runtime.py::test_sample_capture_service_runs_under_repo_venv_with_jack_dependency -q`

2. The live boot script best-effort starts `cypherclaw-sample-capture.service`
   so the daemon is brought up during the standard CypherClaw boot flow.
   VERIFY: `pytest tests/test_cypherclaw_boot_runtime.py::test_boot_script_starts_sample_capture_service -q`

3. The focused service-wiring regression coverage passes end to end.
   VERIFY: `pytest tests/test_cypherclaw_boot_runtime.py -q`
