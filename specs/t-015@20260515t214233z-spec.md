# T-015@20260515T214233Z Specification

## Problem Statement

PAL operators need one command that validates the Phase 1 router after a restart
or instance boot. Existing commands can check health, query PAL, run smoke, and
collect local/SSH diagnostics separately, but there is no single reproducible
workflow artifact that proves all restart validation checks were attempted.

This task implements the PAL-020 `restart_validation` workflow from
`sdp/prd-pal-2026-agentic-ops-platform.md`.

## Technical Approach

- Add a deterministic local workflow in `promptclaw/pal_agent.py` named
  `run_pal_restart_validation(...)`.
- Expose the workflow as:

  ```bash
  promptclaw pal validate restart PROJECT_ROOT
  ```

- The workflow uses fixed local allow-listed tools only:
  - `pal_health`: configured router `/health`.
  - `pal_direct_query`: one fixed direct `/query` prompt.
  - `pal_smoke`: active PAL smoke run, persisted under
    `.promptclaw/pal-smoke/`.
  - `tailscale_status`: local Tailscale status probe.
  - `ssh_process_check`: fixed read-only SSH process/log check, skipped when
    PAL SSH environment variables are absent.
- Write standard run artifacts under `.promptclaw/runs/<run-id>/`:
  - `outputs/restart-validation.json`
  - `routing/route.json`
  - `routing/route.md`
  - `handoffs/restart-validation.md`
  - `summary/final-summary.md`
  - `logs/events.jsonl`
  - `state.json`
- Record `workflow_id: restart_validation`, `mutating_actions: []`, executed
  tool names, per-tool observations, and derived `validation_status`.
- Treat the command as read-only except for local PromptClaw artifacts and local
  smoke report storage.

## Edge Cases

- Missing Tailscale CLI records a skipped/warn-style observation; the workflow
  still completes and records the evidence gap.
- Missing PAL SSH environment variables make the process check record
  `skipped`; no prompt or secret lookup is attempted.
- A failed direct query, failed smoke run, or command exception is captured as
  an error observation through the existing `_run_tool(...)` wrapper.
- Smoke reports are written through the existing PAL smoke writer and no remote
  configuration or service restart is attempted.
- The workflow does not expose action ids and cannot approve, restart, stop,
  rent, destroy, rotate credentials, or change shutdown behavior.
- The generated startup hardening bullets remain regression anchors for the
  existing identity startup subsystem; this PAL validation workflow does not
  alter startup identity behavior.
