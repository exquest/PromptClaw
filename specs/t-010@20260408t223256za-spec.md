# Spec: T-010@20260408T223256Za

## Problem Statement

The live daemon has direct Ollama invocation and basic low-level health helpers, but it does not yet expose one daemon-local helper that summarizes both configured Ollama sockets. This subtask needs a synchronous `ollama_health()` helper in the daemon that queries ports `11434` and `11435` and returns stable per-instance status, loaded model names, and measured response latency for later `/status` and `/local` integration.

## Technical Approach

- Add a daemon-local `ollama_health()` helper in `my-claw/tools/cypherclaw_daemon.py`.
- Probe both Ollama instances in a fixed order: `11434`, then `11435`.
- Reuse the existing `my-claw/tools/ollama_health.py` helpers for health and model discovery instead of introducing a new HTTP client or dependency.
- Measure per-instance latency in the daemon with a wall-clock timer around the health probe.
- Return a JSON-serializable structure with:
  - top-level aggregate `healthy`
  - ordered per-instance records containing `socket`, `port`, `healthy`, `status`, `models`, and `latency_ms`
- Keep the implementation synchronous and side-effect free so it matches existing daemon helper patterns and remains easy to unit test.

## Edge Cases

- If one port is unreachable, that instance should be marked unhealthy without failing the full helper.
- If an instance responds but has no loaded models, it should still be marked healthy with an empty `models` list.
- If both instances are unreachable, the helper should return `healthy: false` with two unhealthy instance entries.
- Latency reporting must not raise on probe failures; failed probes return `latency_ms: null`.

## Acceptance Criteria

1. `cypherclaw_daemon.ollama_health()` returns a stable JSON-serializable summary for both Ollama sockets `11434` and `11435`.
   VERIFY: `pytest tests/test_ollama_daemon_health.py -q`

2. Each per-socket summary includes status, loaded model names, and latency derived from the daemon probe path.
   VERIFY: `pytest tests/test_ollama_daemon_health.py -q`

3. A single-socket or dual-socket probe failure degrades gracefully to unhealthy entries instead of raising exceptions.
   VERIFY: `pytest tests/test_ollama_daemon_health.py -q`
