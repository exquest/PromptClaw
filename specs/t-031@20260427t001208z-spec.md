# Task T-031: Weekly KL-divergence audit between current IDyOM LTM and a frozen week-0 snapshot

## Problem Statement
Once we begin training the IDyOM long-term model (LTM) on generated audio
that flows back through the composer, we run the risk of *generation
collapse*: the LTM gradually narrows around its own outputs, losing
coverage of melodic n-grams seen at week 0 and producing increasingly
self-similar audio. We need a weekly audit job that compares the
current LTM against an immutable week-0 seed snapshot, tracks an
8-week history of three correlated signals, and raises a single,
operator-actionable alert file when all three signals jointly indicate
collapse-drift. The audit must never auto-rollback the LTM — operators
keep that decision.

## Technical Approach
- Add `my-claw/tools/senseweave/generation/health.py` exposing
  `idyom_kl_divergence_audit(...) -> AuditReport` plus a frozen
  `AuditReport` dataclass and a frozen `AuditEntry` history element.
- Represent an LTM as a JSON dict mapping the textual n-gram (joined by
  `,`) to a non-negative count. The audit Laplace-smooths both sides
  (alpha = 1.0) over the union of test n-grams plus an "other" mass
  bucket, normalises to probabilities, and computes symmetric KL as
  `0.5 * (KL(P||Q) + KL(Q||P))` in nats. The "test set" is supplied as
  a sequence of n-gram tuples so callers can pin a fixed set across
  weeks.
- Persist the rolling 8-week history (most-recent-last) at
  `DEFAULT_HISTORY_PATH = /home/user/cypherclaw-data/state/idyom_kl_audit.json`.
  The audit reads the existing history (if any), appends the current
  week's entry with a monotonically-increasing `week_index`, truncates
  to the most recent `history_window` entries (default 8), and writes
  it back atomically via `os.replace`.
- Collapse-drift is flagged only when **all three** signals agree over
  the visible history (need at least 3 points):
    1. KL is increasing — positive linear-regression slope across the
       history's `kl_divergence` series.
    2. `generated_ratio` is high — the latest sample is at or above
       `generated_ratio_threshold` (default `0.5`).
    3. CLAP-centroid variance is decreasing — negative linear-regression
       slope across the history's `clap_centroid_variance` series.
- On flag, write a JSON alert file to
  `/tmp/generation_collapse_alert.json` containing the report's three
  current signal values, the flag reason, and the persisted history.
  Do not modify the LTM, the snapshot, or any composer state — the alert
  is informational and the operator decides whether to roll back.
- The job is intended to run weekly via a systemd `OnCalendar=weekly`
  timer. Add `my-claw/systemd/cypherclaw-idyom-kl-audit.service` and
  `my-claw/systemd/cypherclaw-idyom-kl-audit.timer` so the unit can be
  installed alongside the existing `cypherclaw-*.service` files.
- Pure-stdlib + numpy implementation (numpy is already a transitive
  dependency via `librosa`). No new top-level deps.

## Edge Cases
- Missing history file: treat as empty history; the first run records
  week 0 and never flags (only one data point).
- Snapshot file missing: raise `FileNotFoundError` — the seed snapshot
  is supposed to be immutable; its absence is operationally serious and
  must surface, not silently degrade.
- Current LTM file missing: same — raise `FileNotFoundError`.
- All counts zero in either side: smoothing keeps probabilities defined,
  so KL is finite and the audit still records a value.
- Empty test n-gram set: raise `ValueError("empty test set")`.
- `generated_ratio` outside `[0, 1]` or `clap_centroid_variance` < 0:
  raise `ValueError` with a clear message — these are normalized inputs
  the caller must produce upstream.
- History with fewer than 3 entries: never flag, regardless of signal
  values; record the entry and return `flagged=False` with reason
  `"insufficient history"`.
- All three signals must agree to flag. If KL is increasing and
  `generated_ratio` is high but CLAP variance is increasing or flat,
  the audit reports `flagged=False`.
- `history_path` parent directory does not exist: created on write
  (`mkdir(parents=True, exist_ok=True)`).
- Alert file is overwritten on each flagged run; absence of alert file
  on an unflagged run is the steady-state.

## Acceptance Criteria

- `senseweave.generation.health.idyom_kl_divergence_audit(...) -> AuditReport`
  exists with the documented signature and return type.
  - **VERIFY:** `grep -nE "def idyom_kl_divergence_audit\(" my-claw/tools/senseweave/generation/health.py`

- Symmetric KL is computed over the supplied fixed test set of melodic
  n-grams and is `0.0` for identical LTMs, strictly positive for
  divergent LTMs.
  - **VERIFY:** `pytest tests/test_generation_health.py::test_kl_zero_when_identical -v`
  - **VERIFY:** `pytest tests/test_generation_health.py::test_kl_positive_when_divergent -v`

  the supplied paths and raises `FileNotFoundError` when either is

  - **VERIFY:** `pytest tests/test_generation_health.py::test_missing_snapshot_raises -v`
  - **VERIFY:** `pytest tests/test_generation_health.py::test_missing_current_raises -v`

  disk, and the in-memory `report.history` reflects the post-write

  - **VERIFY:** `pytest tests/test_generation_health.py::test_history_caps_at_eight -v`
