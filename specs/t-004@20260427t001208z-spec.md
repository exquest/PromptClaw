# Task T-004: Provide LocalAdaClient stub

## Problem Statement
The generation subsystem needs a local Ada GPU backend placeholder so callers can
select the future local path without accidentally invoking unfinished hardware
integration. The stub must conform to the existing generation client surface and
fail with a clear operator-facing message until the GPU path is installed.

## Technical Approach
- Add `my-claw/tools/senseweave/generation/client_local.py`.
- Define a `GenerationClient` Protocol with the existing client signature:
  `generate(self, request: Any) -> Any`.
- Define `LocalAdaClient.generate(request)` with the same signature as
  `ReplicateClient.generate` and `ModalClient.generate`.
- Keep the implementation intentionally minimal: always raise
  `NotImplementedError("LocalAdaClient: GPU not yet installed; use Replicate or Modal")`.
- Add focused tests in `tests/test_client_local.py` before implementation.
- Use mypy against the new module and test file as the static Protocol
  satisfaction check because the repo's required `mypy src/` target does not
  include `my-claw/tools`.

## Edge Cases
- Any request payload type must be accepted structurally, because the current
  generation clients use `Any` until typed generation request/result models
  land.
- The local client must not import GPU libraries, probe hardware, or require new
  dependencies.
- The generated startup hardening checks for `bootstrap_identity()` are outside
  this generation-client stub; existing startup identity tests remain the
  verification anchor.

## Acceptance Criteria
- `client_local.py` defines `LocalAdaClient` satisfying the `GenerationClient`
  Protocol signature.
  - **VERIFY:** `MYPYPATH=my-claw/tools python -m mypy my-claw/tools/senseweave/generation/client_local.py tests/test_client_local.py`
- `generate()` raises `NotImplementedError("LocalAdaClient: GPU not yet installed; use Replicate or Modal")`.
  - **VERIFY:** `pytest tests/test_client_local.py::test_local_ada_client_raises_gpu_not_installed_message -q`
- Static type-check confirms Protocol satisfaction.
  - **VERIFY:** `MYPYPATH=my-claw/tools python -m mypy my-claw/tools/senseweave/generation/client_local.py tests/test_client_local.py`
- Test asserts the raise and exact message.
  - **VERIFY:** `pytest tests/test_client_local.py -q`
- Existing startup identity hardening remains green without broadening this task.
  - **VERIFY:** `pytest tests/test_first_boot.py tests/test_governor_integration.py -q`
