# Task T-008@20260427T001208Zc: Generation Queue Claim Transitions

## Problem Statement

The generation queue currently creates the SQLite `queue_items` table and
supports idempotent `enqueue(payload, idempotency_key)`, but workers have no
safe way to claim queued work or return it to the queue after transient
failure. This leaves the future generation worker unable to process items
oldest-first, retry failed attempts, or prevent two workers from claiming the
same row.

## Technical Approach

- Extend `my-claw/tools/senseweave/generation/queue.py` in place.
- Preserve the existing schema and status spelling from T-008a/b:
  `status = "queued"` is the current pending state.
- Add a small typed row model for claimed queue items.
- Implement `GenerationQueue.dequeue()` as one SQLite transaction that:
  - selects the oldest `queued` row by `id`;
  - atomically marks it `running`;
  - increments `attempts`;
  - refreshes `updated_at`;
  - returns the claimed row, or `None` when no work is queued.
- Implement `mark_done(item_id)` for `running -> done`, refreshing
  `updated_at`.
- Implement `mark_failed(item_id)` for `running -> queued` while attempts are
  below `max_attempts`, and `running -> failed` once the retry budget is
  exhausted. Attempts are incremented only when work is claimed by
  `dequeue()`.
- Keep the implementation stdlib-only; no migrations or new dependencies are
  required because the current table already includes `status`, `attempts`, and
  timestamp columns.

## Edge Cases

- Empty queue: `dequeue()` returns `None` without mutating the database.
- FIFO: when multiple rows are queued, the lowest `id` is claimed first.
- Retry: a failed first attempt returns to `queued`, and a subsequent dequeue
  increments attempts again.
- Retry exhaustion: an item whose attempt count has reached `max_attempts`
  transitions to `failed` instead of requeueing.
- No double claim: concurrent `dequeue()` calls against one queued item cannot
  both receive that row.
- Terminal rows: `done` and `failed` rows are never claimed by `dequeue()`.
- Timestamp refresh: `dequeue()`, `mark_done()`, and `mark_failed()` all update
  `updated_at`.
