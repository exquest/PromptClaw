# T-021@20260530T002730Z Spec: Asset Image Render CLI

## Problem Statement

The Deniable Asset Bus producer can now construct argv-safe renderer commands
and ship them to the CypherClaw box, but the image renderer entrypoint itself
does not exist in this repository. The box-side command
`tools/asset_render_image.py` needs a small, documented argv contract so the
producer can request generated PNG assets without knowing DreamShaper internals.

## Technical Approach

- Add `tools/asset_render_image.py` as a thin box-side CLI wrapper.
- Parse the documented argv into a typed `ImageRenderParams` dataclass:
  `--prompt`, `--size WIDTHxHEIGHT`, optional `--seed`, `--count`, and
  `--output-dir`.
- Keep parsing separate from rendering so smoke tests can assert resolved
  render parameters without loading GPU libraries.
- Render `count` PNG files named `image-0.png`, `image-1.png`, ... under
  `output_dir`, creating the directory if needed.
- Use a replaceable renderer callback for tests. The default renderer imports
  the existing DreamShaper pipeline from `my-claw/tools/diffusion_art.py` and
  calls its `generate(...)` function with the resolved prompt/size.
- When a seed is provided, seed Python's `random` module and, if installed,
  `torch` before each generated image with `seed + index`.
- Do not add dependencies, provider secrets, database columns, migrations,
  startup-flow changes, or asset-bus wire-contract changes.

## Edge Cases

- `--size` must be exactly `<positive-int>x<positive-int>` or
  `<positive-int>X<positive-int>`; malformed, zero, or negative dimensions are
  parser errors.
- `--count` must be positive.
- `--seed` is optional. When absent, render parameters carry `None` and no
  deterministic seeding is attempted.
- Output directory can be missing; it is created before writes.
- A renderer that returns a path is copied to the requested output filename if
  it did not write the target path directly.
- Renderer output must be a PNG path or a PIL-like image object with
  `save(...)`.
- The SuperCollider hardening anchors are unrelated to image rendering but are
  explicitly verified for this task because recurring verifier feedback calls
  out `fx_bus_id` routing and `sw_sampler.scd` naming regressions.

## Acceptance Criteria

1. `tools/asset_render_image.py` exposes a typed parser for the documented argv
   and resolves `--prompt`, `--size`, `--seed`, `--count`, and `--output-dir`
   into `ImageRenderParams`.
   VERIFY: `pytest tests/test_asset_render_image.py::test_parse_documented_argv_resolves_render_parameters -q`

2. The CLI render path writes one PNG per requested image into the output
   directory using stable names `image-0.png`, `image-1.png`, ... .
   VERIFY: `pytest tests/test_asset_render_image.py::test_render_images_writes_counted_pngs_with_injected_renderer -q`

3. Invalid sizes and non-positive counts fail during argument parsing before
   any renderer is called.
   VERIFY: `pytest tests/test_asset_render_image.py::test_parse_rejects_invalid_size_and_count -q`

4. The default renderer remains a thin wrapper around the existing
   DreamShaper/Diffusers pipeline and does not introduce new Python
   dependencies.
   VERIFY: `python - <<'PY'\nfrom pathlib import Path\ntext = Path('tools/asset_render_image.py').read_text()\nassert 'diffusion_art' in text\nassert 'from_pretrained' not in text\nPY`

5. Required hardening anchors for recurring verifier feedback still pass:
   SuperCollider voice synthdefs declare `fx_bus_id`, and `sw_sampler.scd`
   uses `fx_bus_id` for its send bus contract.
   VERIFY: `pytest tests/test_space_reverb_profiles.py::test_voice_synthdefs_declare_fx_bus_id_routing_contract tests/test_sw_sampler.py::TestRoutingAndFxSend -q`

6. Product-facing task notes mention the new image render CLI, and no new
   dependencies or migrations are recorded.
   VERIFY: `rg -n "asset_render_image|T-021@20260530T002730Z" CHANGELOG.md progress.md ESCALATIONS.md`

7. The required full validation gate passes.
   VERIFY: `pip install -e '.[dev]' && pytest tests/ -x && ruff check src/ tests/ && mypy src/`
