# Specification: T-016@20260425T183959Zd

## Problem Statement

Subtasks `Za` through `Zc` covered the narrow `select_cast_ids(...)` sampler
hook in `my-claw/tools/senseweave/cast_planner.py`:

- the planner can detect `sw_sampler` in the chosen cast
- it can call a selector-like object
- it can attach a minimal sample summary to mutable cast metadata

What is still missing is the piece-level seam needed to verify the feature the
way the sampler PRD describes it. There is no helper that:

- accepts a fixture piece / piece-like context
- derives selector kwargs (`mode`, `arc_phase`, `mood`, `target_character`)
- returns assembled cast entries, not just bare IDs
- carries the pre-selected sampler metadata onto the assembled cast

Without that seam, the sampler preselection path remains unit-tested in
isolation but not verifiable as a piece-planning operation.

## Technical Approach

1. Extend `my-claw/tools/senseweave/cast_planner.py` with a small,
   piece-aware helper around `select_cast_ids(...)`:
   - accept a piece fixture / piece-like mapping
   - derive selector kwargs from that piece when available
   - return assembled cast entry dicts plus cast metadata
2. Keep the selector dependency Protocol-based:
   - do not introduce a new `SampleSelector` implementation in this task
   - continue accepting stub or real selector-like objects with
     `select(**kwargs)`
3. Preserve the existing metadata contract:
   - keep `cast_metadata["sampler_sample"]` as the minimal summary surface
   - also attach the same sample summary onto sampler cast entries so the
     assembled cast is self-describing during verification
4. Harden the sample summarization path so capture-shaped sample objects that
   lack `duration_sec` can still emit a stable duration using
   `frame_count / sample_rate`
5. Add focused regression tests in `tests/test_cast_planner.py` for:
   - piece-aware cast assembly with `sw_sampler`
   - duration derivation from a capture-shaped record
   - the full `cast_planner` + `sampler_dispatch` targeted verification run

## Edge Cases

- If the piece provides no sampler-relevant fields, piece-derived selector
  kwargs should be empty rather than guessed from unrelated data.
- Explicit `sample_select_kwargs` should override any piece-derived defaults.
- Non-sampler casts must remain unchanged and must not emit sampler metadata.
- Capture-shaped sample records may expose `sample_id`, `path`, `source`,
  `frame_count`, and `sample_rate` but no `duration_sec`; summarization must
  still yield a deterministic duration.
- The assembled cast helper must preserve the existing entry shape used by
  `duet_composer` (`id`, voice fields, `char_name`) so it can be adopted
  without changing downstream tracker assumptions.

## Assumptions

- This checkout still does not contain the concrete `SampleSelector` /
  `sample_library.py` implementation referenced by the sampler PRD, so this
  task intentionally stops at the cast-planner seam and uses Protocol-compatible
  selector stubs in tests.
- `target_character` is represented as a tuple of strings. When a piece does
  not provide it explicitly, falling back to `(patch_name,)` is an acceptable
  bridge for piece-level sampler selection hints in the current tree.
- No new dependencies, migrations, or orchestration-doc updates are required.

## Acceptance Criteria

1. A piece-aware planner helper can assemble a cast for a fixture piece that
   includes `sw_sampler`, and the returned cast metadata contains the
   pre-selected sample.
   VERIFY: `pytest tests/test_cast_planner.py::test_assemble_cast_plans_piece_with_sampler_metadata -q`

2. Sampler metadata summarization derives `duration_sec` from capture-shaped
   records that do not expose a direct duration field.
   VERIFY: `pytest tests/test_cast_planner.py::test_sampler_summary_derives_duration_from_capture_shape -q`

3. The targeted cast planner and sampler dispatch suites pass together after
   the planner seam is added.
   VERIFY: `pytest tests/test_cast_planner.py tests/test_sampler_dispatch.py -q`
