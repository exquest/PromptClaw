# Spec: T-010@20260408T223256Zc

## Problem Statement

`ollama_health()` is now available in the daemon and visible in Telegram `/status`, but operators still do not have the dedicated Telegram `/local` command promised by the R750 local-LLM plan. The current tree also has no shared status JSON endpoint for `/local` to consume. As a result, there is no direct operator command that focuses on local model availability, loaded models, and Ollama socket health.

## Technical Approach

1. Add a shared daemon status snapshot helper in `my-claw/tools/cypherclaw_daemon.py` that returns the core `/status` counters plus the guarded `ollama_health()` payload.
2. Refactor the existing `/status` command path to read from that shared snapshot instead of assembling counts and Ollama data inline.
3. Add a Telegram `/local` built-in command that formats the shared snapshot's `ollama` payload into a compact local-model status message. Keep the scope to the base `/local` status view only.
4. Extend `CypherClawArt` with a dedicated local-status formatter so Telegram display logic stays centralized with the other operator-facing art/text helpers.
5. Add a runtime-style test in a fresh Python interpreter that exercises `handle_builtin("/local")` with stubbed status data and verifies the rendered text.

## Edge Cases

- If `ollama_health()` raises unexpectedly, both the shared status snapshot and `/local` must degrade to an explicit unavailable message instead of crashing.
- If a healthy socket has no loaded models, `/local` must show `no models loaded`.
- If both sockets are down, `/local` must show each socket as unavailable/unreachable and make it clear that local models are not currently available.
- If the shared snapshot returns an empty Ollama instance list, `/local` must still render a meaningful fallback line.

## Acceptance Criteria

1. The daemon exposes a shared status snapshot that includes the guarded Ollama health payload for downstream command formatters.
   VERIFY: `pytest tests/test_local_command_runtime.py::test_status_snapshot_includes_guarded_ollama_payload -q`

2. Telegram `/local` renders per-socket Ollama health, loaded model names, and latency using the shared status snapshot.
   VERIFY: `pytest tests/test_local_command_runtime.py::test_local_command_displays_ollama_from_status_snapshot -q`

3. Telegram `/local` degrades to explicit unavailable text when Ollama health cannot be collected.
   VERIFY: `pytest tests/test_local_command_runtime.py::test_local_command_displays_unavailable_when_ollama_is_missing -q`

4. A fresh-interpreter runtime test can invoke `handle_builtin("/local")` and receive Ollama health text without importing test-only monkeypatch state from the parent process.
   VERIFY: `pytest tests/test_local_command_runtime.py::test_local_command_runtime_renders_ollama_health -q`

5. Existing `/status` Ollama integration coverage continues to pass after the shared snapshot refactor.
   VERIFY: `pytest tests/test_daemon_status_ollama.py -q`
