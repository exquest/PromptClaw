# T-017@20260515T214233Z Specification

## Problem Statement

PAL operators need a reproducible way to assess whether the current Phase 1
deployment is ready to even discuss Phase 2 multi-GPU / larger-model work.
The PRD explicitly forbids Phase 2 execution in this workstream, so the
workflow must produce a scored readiness report only. It must not expose an
approval action, cloud lifecycle action, model-loading action, volume migration,
or any other Phase 2 execution path.

This task implements the PAL-022 `phase2_readiness_report` workflow from
`sdp/prd-pal-2026-agentic-ops-platform.md`.

## Technical Approach

- Add a deterministic local workflow in `promptclaw/pal_agent.py` named
  `run_pal_phase2_readiness_report(...)`.
- Expose the workflow as:

  ```bash
  promptclaw pal report phase2-readiness PROJECT_ROOT
  ```

- The workflow uses fixed read-only tools only:
  - `pal_health`: configured PAL router `/health`.
  - `pal_smoke_baseline`: saved PAL smoke baseline summary and token/s data.
  - `shutdown_remote_audit`: existing fixed read-only shutdown audit diagnostic.
  - `phase2_project_state`: local project/runbook state check.
  - `vast_boundary`: local Vast connector boundary check.
- Score these prerequisites in a JSON-safe report:
  - `operator_authorization`: always blocked until Anthony explicitly authorizes
    Phase 2 outside this report.
  - `phase1_health_baseline`: router health and smoke baseline evidence.
  - `shutdown_safety`: shutdown audit state and override evidence.
  - `deployment_reproducibility`: local PAL runbook/session-state evidence.
  - `cost_and_vast_boundary`: Vast lifecycle operations remain blocked.
  - `phase2_execution_boundary`: the report exposes no execution actions.
- Write standard run artifacts under `.promptclaw/runs/<run-id>/`:
  - `outputs/phase2-readiness.json`
  - `routing/route.json`
  - `routing/route.md`
  - `handoffs/phase2-readiness.md`
  - `summary/final-summary.md`
  - `logs/events.jsonl`
  - `state.json`
- Record `workflow_id: phase2_readiness_report`, per-prerequisite scores,
  `overall_score`, `readiness_status`, executed tool names,
  `mutating_actions: []`, and `phase2_execution_actions: []`.
- Treat the command as read-only except for local PromptClaw run artifacts.

## Edge Cases

- Missing PAL SSH environment variables make the shutdown audit tool record
  `skipped`; the readiness report still scores shutdown safety with available
  evidence instead of prompting for credentials.
- Missing smoke baselines or non-green router health lower the Phase 1 health
  score but do not fail artifact creation.
- Missing PAL runbook/session-state files lower the deployment reproducibility
  score and are reported as evidence gaps.
- If any fixed diagnostic raises or returns malformed data, `_run_tool(...)`
  records an error observation and the related prerequisite scores low.
- The operator authorization prerequisite remains blocked by default so PAL
  cannot declare Phase 2 ready on technical evidence alone.
- The workflow never rents, starts, stops, destroys, or resizes instances; loads
  Phase 2 models; migrates volumes; edits remote files; restarts services;
  changes shutdown behavior; exposes approval action ids; or calls the PAL
  action registry.
- The generated startup hardening bullets remain regression anchors for the
  existing identity startup subsystem; this PAL readiness report does not alter
  startup identity behavior.
