## Correctness

`log_config.py` correctly implements the required deliverables:

- `setup_logging()` configures structlog with a full JSON/structured processor chain: context-var merging, log level, logger name, ISO timestamp, positional args, stack info, exc_info, unicode decoding, and JSON rendering via `ProcessorFormatter`.
- The uvicorn/uvicorn.error/uvicorn.access/fastapi loggers are all captured and routed through the structured handler with `propagate = False`, preventing duplicate output.
- `StructlogRequestMiddleware` binds `method` and `path` per-request via contextvars, measures wall-clock duration with `perf_counter`, emits `request_finished` (with status_code and duration) on success, and `request_failed` (with duration) on exception before re-raising.
- `app.py` calls `setup_logging()` and adds `StructlogRequestMiddleware` at app creation time — both wired correctly.
- The `if structlog.is_configured(): return` guard is correct — prevents double-configuration across multiple `create_app()` calls in tests.

No behavioral regressions: all 3909 tests pass (`pytest tests/ -x`, 3 skipped, 296 warnings).
