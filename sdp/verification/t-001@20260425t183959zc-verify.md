## Correctness

Implementation is correct. `BufferLoader` uses `OrderedDict[int, BufferRecord]` keyed by `id(record)` for O(1) LRU promotion. Load calls `_evict_lru()` before allocating when `len(self._lru) >= self.capacity`, ensuring the live set never exceeds capacity. `touch()` calls `move_to_end(key)`, correctly promoting to MRU. `on_sampler_free` removes the record from `_lru` so freed records don't count against capacity. `_evict_lru` pops the front (`last=False`) entry, sends `/b_free`, and clears `buffer_id` on the evicted record. Ordering is preserved correctly in all paths.

## Completeness

Five LRU-specific unit tests were added covering: basic eviction on overflow, touch-promoted MRU survives eviction, touch no-op for untracked record, free-then-reload does not trigger spurious eviction, and no eviction below capacity. These cover all task-specified scenarios (ordering and eviction trigger).

Two minor gaps:
- **Double-load edge case**: Calling `on_sampler_load` on a record already live in `_lru` silently overwrites the `_lru` entry with a new bufnum, leaking the old bufnum (never freed, no OSC `/b_free` sent). No test covers this.
- **capacity=0**: `len(self._lru) >= 0` is always true even on first load, so `_evict_lru()` is called on an empty `OrderedDict`, raising `KeyError`. No guard or validation prevents `capacity=0`. Low risk for production but unguarded.
