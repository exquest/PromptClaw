- `NoNewPrivileges`, `ProtectSystem=strict`, `PrivateTmp=true` provide appropriate confinement for a network-calling background worker.
- **Minor concern:** `ProtectSystem=strict` makes all paths read-only except those listed in `ReadWritePaths`. Only `/home/user/cypherclaw-data` is writable; `WorkingDirectory=/home/user/cypherclaw` is **read-only at runtime**. If `generation_worker.py` writes log files, a PID file, or any transient state under its working directory, those writes will fail with `EROFS`. The data path `/home/user/cypherclaw-data` should be sufficient for worker output, but this must be confirmed when the worker script is authored. If the worker writes to the repo dir, `ReadWritePaths=/home/user/cypherclaw` will need to be added (or the writes redirected to the data path).
