## Correctness

Implementation matches the spec precisely. `MotifMemory` is a fixed-capacity ordered dict (`collections.OrderedDict`) storing `MotifMemoryEntry` with `motif_hash`, `timestamp`, `hit_count`, and `decay_weight`. All four required methods are present:

- **insert**: Adds new entry or increments hit_count and moves to end on re-insert; evicts LRU on overflow.
- **lookup**: Updates timestamp, increments hit_count, moves to end; returns `None` on miss.
- **evict**: Pops the least-recently-used (first) entry; returns `None` on empty cache.
- **return_home_decay**: Computes and stores a phase-linear decay weight based on hit_count modulo `return_home_iterations`; resets to 1.0 at cycle boundaries.

The `MotifMemoryRule` (R9) wraps the cache with a pitch-class + rhythm hash, role-gate check, variation-menu selection, and `should_return_home` gating. Logic is correct.
