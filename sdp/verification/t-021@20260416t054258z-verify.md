The spec mentions the planner should support `"spectral"` and `"microtonal"` as tagged strategies, but no code paths exist for those tags. The acceptance-criteria test asserts `plan.strategy in {"post-tonal", "spectral", "microtonal"}` — this passes only because the returned value is `"post-tonal"`.

2. **Planner does not consume the new helpers:** The spec states the planner "can use `PitchClassSet`, `spectral_partials`, or `quarter_tone_to_freq` to bound the generated material." Callers that pass extreme inputs will receive out-of-range frequencies silently.

4. **Spec test-ID mismatch:** `specs/…-spec.md` references `-k "test_post_tonal_and_spectral"` under AC-1, but no test carries that exact name. Tests are named `TestPostTonalAndSpectral::test_*`. The correct filter is `-k "PostTonal or post_tonal"`.
