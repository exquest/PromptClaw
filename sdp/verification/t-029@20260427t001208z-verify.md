## Correctness

All acceptance criteria are satisfied:

- `validate_audio(path) -> ValidationReport` is defined at `my-claw/tools/senseweave/generation/validate.py` (consistent with the project's established `my-claw/tools/senseweave/` layout).
- `ValidationReport` is a dataclass with all required fields: `valid`, `reason`, `peak_dbfs`, `rms_dbfs`, `nan_count`, `format`.
- Rejection conditions are correctly implemented:
  - Clipping: `peak_dbfs > 0.0` → rejected with "clipping" reason.
  - Silence: `rms_dbfs < -50.0` → rejected with "silence" reason. All-zero arrays yield `rms = 0.0` → `_amplitude_to_dbfs(0.0) = -inf`, which is correctly less than -50.
  - NaN: `nan_count > 0` → rejected. NaN samples are excluded from peak/RMS computation via masked indexing, preventing `np.nan` propagation.
  - Format: Only `WAV` and `FLAC` (uppercased from `sf.info().format`) are accepted.
- Mono and stereo are both accepted; channel counts other than 1 or 2 are rejected.
- Sample rate ≥ 22050 Hz enforced.
- Unreadable file returns a `ValidationReport(valid=False)` rather than raising — good defensive posture.
