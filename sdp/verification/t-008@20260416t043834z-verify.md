## Correctness

All acceptance criteria are met:

**Shared state contract:** `normalize_theramini_state()` in `theramini_duet.py` is the single contract consumed by both listeners and the composer. Both `theramini_listener.py:write_state()` and `theramini_midi.py:write_state()` call it before writing to `/tmp/theramini_state.json`. The returned dict always includes `listening`, `speaking`, `silence_request`, `human_gesture_active`, `midi_cc`, and a `conversation` sub-dict with `phase`, `policy`, `speaker`, `may_play`, `duet_active`.

**Policy coverage:** All five policies are implemented:
- `turn_taking` — default while human gesture is active (may_play=False)
- `imitation` — slow onset rate or explicit request
- `counterpoint` — fast onset rate (>2/s) or explicit request
- `accompaniment` — medium onset rate or explicit request
- `silence_request` — triggered by CC (breath/volume/expression ≤ 3) or explicit state flag
- `solo` — recovery mode after `_SOLO_INACTIVITY_MS` (5000ms) of inactivity

**No doubling/crowding:** `plan_duet_response()` checks `_human_gesture_active()` first; if True and no silence request, it returns `may_play=False, phase="listening", speaker="human"`. The composer's `duet_loop()` only calls play functions when `decision.may_play` is True.

**Recovery to solo:** After `consecutive_silence_ms >= 5000` (with stale-timestamp fail-safe at 10s), `plan_duet_response()` returns `duet_active=False, phase="solo"`, and `duet_loop()` breaks out to solo mode.

**Idle tone detection:** `theramini_listener.py` tracks pitch history; if pitch variance < 5Hz over 4+ samples, sets `idle_tone=True` and `is_playing=False`. `_human_gesture_active()` explicitly checks `idle_tone`.
