## Correctness

`event_id`, `phrase_id`, `section_id`, `voice_id`, `role`, `pitch`, `nominal_beat`, `nominal_dur_beats`, `harmonic_charge`, `melodic_charge`, `metric_weight`, `is_phrase_start`, `is_phrase_end`, `is_cadential`, `intent_tag`, `onset_sec`, `dur_sec`, `velocity`, `timing_deviation_ms`, `articulation`, `sensor_tempo_scale`, `sensor_amp_scale`, `sensor_brightness`, `rule_stack`, `seed_path`.

Score-level immutability is enforced via a custom `__setattr__` that checks `_score_fields_locked`. The 15 score-level fields match what the spec describes as "score-level" (pre-render identity and content). Performance-level fields remain mutable after locking.

JSON round-trip works via `to_json_dict()` / `from_json_dict()`. OSC bundle encoding uses a minimal OSC 1.1-compatible format carrying the JSON payload as a string argument, and round-trips correctly. `seed_path` is correctly coerced back to `tuple[int, ...]` after JSON deserialization (JSON arrays become lists; `from_json_dict` re-coerces via `__post_init__`).

## Completeness

- All acceptance criteria fields present and tested.
- Three test functions cover: field presence, immutability enforcement, and full JSON + OSC round-trip.
- `rule_stack` mutability post-lock is tested (`.append` allowed after `lock_score_fields()`), which is the correct behavior since it's a performance-level field.

- **Minor gap:** No test for `section_envelope` round-trip specifically with a non-default value. The dict-reconstruction path in `__post_init__` is exercised only if `section_envelope` arrives as a dict (which happens during `from_json_dict`), but the test event leaves `section_envelope=None`. Functional but uncovered path.
- **Minor gap:** The `_score_fields_locked` attribute is stored via `object.__setattr__` and is not a declared dataclass field, so it is excluded from `to_json_dict()`. This is correct behavior but means lock state does not survive serialization (event reconstructed from JSON starts unlocked). This is intentional and appropriate — caller must re-lock after deserialization — but is not documented.
