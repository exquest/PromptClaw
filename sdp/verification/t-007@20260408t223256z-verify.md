## Completeness

The implementation handles the main failure scenarios:

- Single Ollama port down: excluded.
- All Ollama ports down: excluded.
- All cloud providers also quota-paused AND Ollama unhealthy: `get_available_agents` returns `[]`, `_eligible_agents` returns `[]`, `select` raises `ValueError("no agents available for selection")` — correct escalation behavior.
- Partial port recovery: any healthy port re-admits Ollama (documented tradeoff, see Consistency).

**Minor gap**: The `available = [agent for agent in quota_filtered if agent in filtered]` guard in `_eligible_agents` is logically redundant (inputs guarantee `quota_filtered ⊆ filtered`), but harmless. The consequent `return [] if not quota_filtered else filtered` branch for the `available`-empty-but-`quota_filtered`-non-empty case is unreachable in practice. Not a correctness issue.

**Test coverage**: Two new targeted tests cover exclusion, fallback, and recovery paths at both the `QuotaMonitor` and `AgentSelector` levels. The `autouse` `_healthy_ollama` fixture correctly protects pre-existing tests from the new live health check.
