# PRD: CypherClaw Humanization — Structured Expression Rendering Layer

## Overview

CypherClaw already composes legible music: score trees, section functions, production profiles, genre strategies, SenseWeave mappings. But the path from composed score to synthesized audio still renders too rigidly. Phrases don't breathe. Repeated motifs don't vary. Silence is a byproduct of scheduling rather than an intentional voice. Sensors occasionally read like one-to-one controllers rather than shaping aggregate intent. The result is the "almost-human" failure mode: structurally competent, expressively flat.

The fix is not sprinkled randomness. The research literature is unambiguous on this: human expressive variance is **structured, not stochastic**. Random jitter is what produces the uncanny-valley failure, not what escapes it.

This PRD installs a dedicated **rendering layer** — a compiler pass between composition (score-tree) and synthesis (SuperCollider) — that translates score structure into correlated, intentional deviations in timing, dynamics, articulation, and silence. The architecture is KTH-rule-system-style: a stack of ordered, pure-function rules, each reading score features and emitting event-level deviations, each toggleable and quantity-adjustable, each leaving provenance on every event so authors can ablate and debug.

Five principles govern the whole design:

1. **Structured over stochastic.** Every deviation is a function of score/sensor features. Gaussian jitter is allowed only below the listener JND (~3–5 ms, ~2 velocity units) and only as a final polish *on top of* intentional deviations.
2. **Hierarchical arcs.** Section-level tempo elasticity modulates phrase-level arcs, which modulate note-level accents. Arcs compose multiplicatively.
3. **Legible intent.** Every phrase declares an intent tag (build/settle/question/answer/withhold/punctuate). Parameters are chosen so the intent is perceptible.
4. **Restraint as material.** Silence is scheduled, not residual. Density and silence budgets are maintained per voice and per section.
5. **Machine identity preserved.** Grid-locked elements (ostinati, sequencers, polyrhythmic machines), superhuman precision, and alien timbres are deliberately *not* humanized. CypherClaw is a robot musician that has learned which things humans shape — not a robot pretending to be a violinist.

The goal is not realism. It is **legibility**: a listener should feel that CypherClaw *intended* each phrase — to build, to withhold, to answer the room.

## Scope and non-goals

**In scope:** a new rendering stage between composition and synthesis; an Event/PerformanceIntent/SectionEnvelope data model with rule-stack provenance; a 12-rule render stack (R1–R12); a seeded-RNG architecture with per-stage sub-seeds; a replay harness; a rule-ablation debugger; a metrics-CI gate with anti-pattern detection; a performance-worm diagnostic; a bounded sensor-to-intent aggregation matrix.

**Out of scope / non-goals:**

- Photorealistic human-performance imitation. CypherClaw is a robot musician, not a violinist simulator.
- General-purpose MIDI humanizer — this is not a plugin.
- Sensor-driven note generation. Sensors modulate expression, never replace composition.
- Machine-learning performance models in this phase. Rule-based KTH-style is interpretable, debuggable, and gateable with fixed metrics. Learned models can be added after the metric gate is stable.
- Replacement of existing composition, production-course, or synthesis work. This layer sits *above* synthesis and *below* composition — it renders what the composer emits.

## The four phases

**Phase 1 — Foundations (CCH-001..CCH-010).** Data model, seeded-RNG plumbing, replay harness, rule-ablation debugger. At end of phase, existing pieces render bit-identically through the new pipeline with no rules active — just plumbing.

**Phase 2 — Core rule stack (CCH-011..CCH-021).** R1 Metric Accent, R2 Phrase Arch (Todd/KTH parabolic), R3 Agogic Accent, R4 Duration Contrast, R5 Final Ritardando (Friberg–Sundberg square-root), R6 Punctuation+Breath, R7 Articulation Shaping, R8 Microtiming Polish (below JND). Role-filter gating so grid-locked voices skip the stack. Phrase-feature extraction via partitura. End-of-phase: single-voice melodic passages sound shaped, not flat; metric gate shows structured deviations vs. flat-line baseline.

**Phase 3 — Memory, silence, ensemble (CCH-022..CCH-029).** R9 Motif memory with LRU + variation menu + return-home decay. R10 Silence budgets with wind-voice lung-capacity caps. R11 Call-response gating. Section-level envelope compositing. Tempo-dependent swing (Friberg/Sundström). Arc-shape variety. Restraint as global dampener. End-of-phase: 5-minute generative pieces no longer feel looped; repetition-variance metrics pass thresholds.

**Phase 4 — Sensor aggregation + narrative coupling (CCH-030..CCH-033).** R12 Sensor-to-intent matrix (smoothed IIR, hysteresis, aggregate intentions — never one-to-one). Bounded reactive delta track. Narrative engine → SectionEnvelope coupling. End-of-phase: installation responds legibly to audience presence without gimmicky mappings.

**Metrics-CI Gate + Diagnostics (CCH-034..CCH-040).** Metric battery, anti-pattern detection, performance-worm logger, CI integration, audio-rendered cross-check, provenance in exported audio metadata, listener-review workflow. Runs in parallel with rule-stack build-out; most checks land alongside the rule they validate.

## Acceptance criteria (top-level)

1. End-to-end: existing pieces render bit-identically through the new pipeline when all rules are disabled (Phase 1 plumbing is transparent).
2. With R1–R8 enabled, single-voice melodic passages show structured velocity + timing deviations matching research pass bands (CCH-034).
3. Grid-locked voices (ostinato, perc) emerge from full-stack render with timing_deviation_ms == 0 — machine identity preserved.
4. Motif Memory (R9) produces at least one variation per recurrence and returns to the home motif within ~8 iterations.
5. Silence Budget (R10) enforces ≥100 ms silence at every phrase boundary on melodic voices in a 2-minute rendered fixture.
6. Call-Response Gating (R11) on a 2-voice leader/follower fixture produces complementary phrase shapes (verifiable via phrase-apex correlation < -0.3).
7. Sensor Aggregation (R12) passes the reverse-engineer test: no single sensor correlates > 0.85 with any single synth parameter over a 5-minute rendered run with recorded sensor stream.
8. Replay harness reproduces past performances byte-identically given (score, seeds, delta_track).
9. Rule-ablation debugger produces a human-readable diff of which Event fields changed when a rule is disabled.
10. Metric battery + anti-pattern detectors integrated into CI; a deliberate regression (e.g., microtiming σ bumped to 50 ms, velocity range compressed to 5) fails the gate.
11. Performance-worm PNG is attached to CI artifacts for every fixture piece.
12. Rule-stack provenance present in every exported `.render.json` sidecar.

## Research basis (reference)

- **KTH rule system** (Friberg, Sundberg, Frydén, 1978–2006; Director Musices). Phrase Arch, Duration Contrast, Melodic + Harmonic Charge, Final Ritardando, Punctuation rules with per-rule quantity k.
- **Todd 1985/1992** — parabolic tempo + loudness arcs aligned with phrase structure.
- **Widmer (JKU Linz)** — ML performance models validating "stress the apex of up-down contour" and related local principles.
- **Friberg & Sundström** — ride-cymbal swing ratio 3.5:1 at slow tempi → ~1:1 at fast; short note ~100 ms near-constant.
- **Benadon** — soloist BUR 1.2–1.7 typically less swung than cymbal.
- **Friberg & Sundberg 1995** — listener JND for isochronous perturbation ~10 ms.
- **Senn et al. 2016 / Butterfield 2010** — groove microtiming within ±50 ms, mostly under 30 ms.
- **Lerdahl tonal pitch space** — harmonic charge via pitch-space distance + dissonance + attraction.
- **Huron 2006 (Sweet Anticipation) / Margulis / Farbood 2012** — tension modeling (ITPRA; surprise/denial/expectancy).
- **Goebl & Palmer 2009 / Keller 2008 / Large & Palmer 2002** — ensemble adaptation argues for internal "listening" state even when solo.
- **Savery, Rose & Weinberg 2019 (Shimon)** — prosody-as-robot-identity; commit to machine identity rather than pretend.
- **partitura / Basis Mixer / ACCompanion (CP-JKU, MEC 2022, IJCAI 2023)** — architectural pattern for symbolic-score → performed-part stages.
- **Câmara et al. 2022 / Repp 1992** — expressive variance is structured, not stochastic.

## Anti-patterns to reject (built into CI)

- **Random-as-expression.** Gaussian jitter without structured deviation. Ljung-Box must reject white-noise null on tempo and velocity series.
- **Enthusiastic rule application.** Doubling every k produces caricature, not more expression. Defaults are calibrated and per-piece overrides must be justified.
- **Forgetting machine identity.** If a rule leaks onto grid-locked voices, the render fails the role-filter gate.
- **One-to-one sensor gimmickry.** No listener-reverse-engineerable sensor → parameter mapping.
- **Symmetric-arc uncanny.** Every phrase cannot have the same parabolic shape.
- **Permanent novelty / no return home.** Motif memory must bring the original back within ~8 iterations.
- **Dead-silence failure.** A 2-minute melodic passage with no rests ≥200 ms fails — silence is a first-class voice.
