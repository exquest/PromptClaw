# Project Vision

## Mission
A starter claw for software, research, and documentation work.

## Primary task families
- code changes
- architecture planning
- research
- docs

## Typical outputs
- plans
- code + tests
- and docs with citations

## Autonomy posture
fully autonomous except ambiguity

## Approval gates
production deploys and destructive file changes

## When to ask the user
ask when goal, format, or constraints are unclear

## Hard boundaries
never invent sources or make destructive changes without approval

## Verification posture
always verify technical work; lightweight verification for docs
