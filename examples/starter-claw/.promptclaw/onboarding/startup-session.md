# Startup Session

## Question 1
**Prompt:** What kind of PromptClaw are we building?

**Answer:** A starter claw for software, research, and documentation work.

## Question 2
**Prompt:** What are the top task families it should handle?

**Answer:** code changes, architecture planning, research, docs

## Question 3
**Prompt:** What should it usually produce for the user?

**Answer:** plans, code + tests, and docs with citations

## Question 4
**Prompt:** Who is on the claw team?

**Answer:** codex, claude, gemini

## Question 5
**Prompt:** What is codex best at in this claw?

**Answer:** implementation, tests, and refactors

## Question 6
**Prompt:** What is claude best at in this claw?

**Answer:** architecture, specs, and verification

## Question 7
**Prompt:** What is gemini best at in this claw?

**Answer:** research, synthesis, and documentation

## Question 8
**Prompt:** How should the orchestrator choose a lead agent?

**Answer:** route code to codex, architecture to claude, and research/docs to gemini

## Question 9
**Prompt:** What is the verification style?

**Answer:** always verify technical work; lightweight verification for docs

## Question 10
**Prompt:** How autonomous should the claw be once it starts?

**Answer:** fully autonomous except ambiguity

## Follow-up 11
**Prompt:** Name any actions that should still require explicit user approval.

**Answer:** production deploys and destructive file changes

## Question 12
**Prompt:** When should it stop and ask the user a question?

**Answer:** ask when goal, format, or constraints are unclear

## Question 13
**Prompt:** What should it never do?

**Answer:** never invent sources or make destructive changes without approval
